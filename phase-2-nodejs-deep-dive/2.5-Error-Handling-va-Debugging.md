# §2.5 — Error Handling & Debugging

> **Mục tiêu**: Xử lý errors đúng cách trong Node.js — operational vs programmer errors, async error handling, debugging tools, memory leak detection, performance profiling.

---

## 1. Error Types

```javascript
// OPERATIONAL errors — expected, recoverable
// DB connection fail, file not found, API timeout, validation fail
// → Handle gracefully (retry, fallback, user message)

// PROGRAMMER errors — bugs, unexpected
// TypeError, undefined access, wrong logic
// → Fix code, NOT handle at runtime

class AppError extends Error {
  constructor(message, statusCode, isOperational = true) {
    super(message);
    this.statusCode = statusCode;
    this.isOperational = isOperational;
    Error.captureStackTrace(this, this.constructor);
  }
}

class NotFoundError extends AppError {
  constructor(resource) {
    super(`${resource} not found`, 404);
  }
}

class ValidationError extends AppError {
  constructor(details) {
    super("Validation failed", 422);
    this.details = details;
  }
}
```

## 2. Async Error Handling

```javascript
// ⚠️ unhandledRejection — Promise rejected without .catch()
process.on("unhandledRejection", (reason, promise) => {
  console.error("Unhandled Rejection:", reason);
  // Log to monitoring → graceful shutdown
  process.exit(1); // Node.js v15+: crashes by default!
});

// ⚠️ uncaughtException — sync throw without try/catch
process.on("uncaughtException", (err) => {
  console.error("Uncaught Exception:", err);
  // ⚠️ Process state is UNRELIABLE → MUST exit!
  process.exit(1);
});

// Express global error handler:
app.use((err, req, res, next) => {
  if (err.isOperational) {
    res.status(err.statusCode).json({ error: err.message });
  } else {
    console.error("PROGRAMMER ERROR:", err);
    res.status(500).json({ error: "Internal Server Error" });
    process.exit(1); // unsafe state → restart via PM2/Docker
  }
});
```

## 3. Debugging Tools

```bash
# Chrome DevTools
node --inspect app.js        # open chrome://inspect
node --inspect-brk app.js    # break on first line

# CPU Profiling
node --prof app.js            # generate V8 log
node --prof-process isolate-*.log > processed.txt

# Heap Snapshot
node --expose-gc              # allow manual GC
```

```javascript
// Performance measurement
const { performance, PerformanceObserver } = require("perf_hooks");

performance.mark("start");
// ... code to measure ...
performance.mark("end");
performance.measure("API Call", "start", "end");

// Memory Usage
console.log(process.memoryUsage());
// { rss: 30MB, heapTotal: 10MB, heapUsed: 7MB, external: 1MB }
// rss = total allocated | heapUsed = actual objects | external = C++ objects
```

## 4. Tools Summary

| Tool                   | Use                            |
| ---------------------- | ------------------------------ |
| `--inspect`            | Chrome DevTools debugger       |
| `clinic flame`         | Flame graphs (CPU bottlenecks) |
| `clinic doctor`        | Automated diagnostics          |
| `0x`                   | Flame graph generation         |
| `autocannon`           | HTTP load testing              |
| `heapdump`             | Heap snapshot for memory leaks |
| `--max-old-space-size` | Increase heap limit            |
