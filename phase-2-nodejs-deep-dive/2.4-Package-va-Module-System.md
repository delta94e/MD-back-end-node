# §2.4 — Package & Module System

> **Mục tiêu**: Hiểu require() resolution algorithm, ESM interop, npm workings, và security.

---

## 1. require() Resolution Algorithm

```
require("X") — Node.js tìm module thế nào?

1. Core module? (fs, http, path...) → load từ Node.js built-in
2. Starts with "./" or "../" or "/"?
   → File: X.js → X.json → X.node
   → Directory: X/package.json "main" → X/index.js
3. node_modules lookup:
   ./node_modules/X
   ../node_modules/X
   ../../node_modules/X
   ... (lên đến root /)

Module caching:
  require("X") lần 1: load + execute + CACHE
  require("X") lần 2: return CACHED module (không execute lại!)
  → Singleton pattern tự nhiên
```

## 2. Circular Dependencies

```javascript
// a.js
exports.loaded = false;
const b = require("./b"); // b require a → a trả PARTIAL exports!
exports.loaded = true;

// b.js
const a = require("./a");
console.log(a.loaded); // false! (a chưa chạy xong khi b require a)

// ESM khác: live bindings → luôn nhận giá trị mới nhất
```

## 3. npm Deep Dive

```json
// package.json critical fields:
{
  "name": "my-app",
  "version": "1.2.3", // semver: major.minor.patch
  "main": "./dist/index.js", // CJS entry point
  "module": "./dist/index.mjs", // ESM entry point
  "type": "module", // ESM by default

  "dependencies": {}, // production
  "devDependencies": {}, // development only
  "peerDependencies": {
    // "YOU must install this"
    "react": ">=17.0.0"
  },
  "engines": { "node": ">=18" },
  "scripts": {
    "start": "node dist/index.js",
    "dev": "tsx watch src/index.ts"
  }
}

// Semver: ^1.2.3 = >=1.2.3 <2.0.0 (minor + patch updates)
//         ~1.2.3 = >=1.2.3 <1.3.0 (patch updates only)
// package-lock.json: LOCK exact versions → reproducible builds
// ⚠️ LUÔN commit package-lock.json!
```
