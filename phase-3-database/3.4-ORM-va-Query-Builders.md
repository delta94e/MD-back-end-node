# §3.4 — ORM & Query Builders

> **Mục tiêu**: So sánh Prisma, TypeORM, Knex.js. Biết khi nào dùng ORM, khi nào raw SQL. Type safety, migrations, performance.

---

## 1. Prisma

```prisma
// schema.prisma — type-safe schema definition
model User {
  id        Int      @id @default(autoincrement())
  email     String   @unique
  name      String?
  posts     Post[]
  createdAt DateTime @default(now())
}

model Post {
  id       Int    @id @default(autoincrement())
  title    String
  author   User   @relation(fields: [authorId], references: [id])
  authorId Int
}
```

```typescript
// Type-safe queries — autocomplete + compile-time check!
const user = await prisma.user.findUnique({
  where: { email: "alice@example.com" },
  include: { posts: true }, // eager load relations
});

// Transaction
await prisma.$transaction([
  prisma.account.update({
    where: { id: fromId },
    data: { balance: { decrement: 100 } },
  }),
  prisma.account.update({
    where: { id: toId },
    data: { balance: { increment: 100 } },
  }),
]);

// Raw SQL khi cần
const result = await prisma.$queryRaw`
  SELECT * FROM users WHERE email = ${email}
`; // parameterized → safe from SQL injection!

// Migrations
// npx prisma migrate dev --name add_posts_table
// npx prisma migrate deploy (production)
```

## 2. TypeORM

```typescript
// Entity — Active Record hoặc Data Mapper patterns
@Entity()
export class User {
  @PrimaryGeneratedColumn()
  id: number;

  @Column({ unique: true })
  email: string;

  @OneToMany(() => Post, (post) => post.author)
  posts: Post[];
}

// Data Mapper (preferred cho large apps)
const user = await userRepository.findOne({
  where: { id: 1 },
  relations: ["posts"],
});

// Query Builder
const users = await userRepository
  .createQueryBuilder("user")
  .leftJoinAndSelect("user.posts", "post")
  .where("user.email LIKE :email", { email: "%@example.com" })
  .orderBy("user.createdAt", "DESC")
  .take(10)
  .getMany();
```

## 3. Knex.js — Query Builder

```typescript
// Query builder — SQL gần hơn, linh hoạt hơn ORM
const users = await knex("users")
  .select("users.name", knex.raw("COUNT(orders.id) as order_count"))
  .leftJoin("orders", "users.id", "orders.user_id")
  .groupBy("users.id")
  .having("order_count", ">", 5)
  .orderBy("order_count", "desc");

// Migrations
// knex migrate:make create_users_table
exports.up = (knex) =>
  knex.schema.createTable("users", (t) => {
    t.increments("id").primary();
    t.string("email").unique().notNullable();
    t.timestamps(true, true);
  });
```

## 4. So Sánh & Khi Nào Raw SQL

| Feature         | Prisma      | TypeORM         | Knex.js     | Raw SQL      |
| --------------- | ----------- | --------------- | ----------- | ------------ |
| Type Safety     | ✅ Best     | ✅ Good         | ⚠️ Manual   | ❌           |
| Learning Curve  | Thấp        | Trung bình      | Thấp        | Cao          |
| Complex Queries | ⚠️ Limited  | ✅ QueryBuilder | ✅          | ✅ Best      |
| Performance     | Tốt         | Tốt             | Tốt         | **Tốt nhất** |
| Migrations      | ✅ Built-in | ✅ Built-in     | ✅ Built-in | Manual       |

```
Khi nào Raw SQL?
  ✅ Window functions phức tạp
  ✅ Recursive CTEs
  ✅ Database-specific features (JSONB, full-text)
  ✅ Performance-critical queries
  ✅ Complex JOINs + aggregations

  ⚠️ LUÔN dùng parameterized queries → chống SQL injection!
  ❌ KHÔNG concatenate user input vào SQL string!
```
