## 1. Array — Mảng

### 1.1 Array là gì — Từ mức Memory

Array là **tập hợp phần tử cùng kiểu** được lưu **liên tiếp nhau** (contiguous) trong memory.

```
Memory (mỗi ô = 4 bytes cho int32):

Address:   0x100  0x104  0x108  0x10C  0x110
           ┌──────┬──────┬──────┬──────┬──────┐
arr[]:     │  10  │  20  │  30  │  40  │  50  │
           └──────┴──────┴──────┴──────┴──────┘
Index:       [0]    [1]    [2]    [3]    [4]

Truy cập arr[3]:
  address = base_address + (index × element_size)
  address = 0x100 + (3 × 4) = 0x10C
  → O(1) — chỉ 1 phép tính, không cần duyệt!
```

**Tại sao O(1) access?** Vì CPU tính trực tiếp địa chỉ bằng công thức — không cần lần theo con trỏ như Linked List.

**Tại sao cache-friendly?** Contiguous memory → CPU cache load cả block liền kề → spatial locality → ít cache miss (xem §0.1.2).

### 1.2 Static Array vs Dynamic Array

```
Static Array (C/Java):
  Kích thước cố định khi tạo. Không thể thay đổi.
  int arr[5]; // 5 phần tử, không thể thêm

Dynamic Array (JS Array, Python List, C++ vector):
  Tự động mở rộng khi cần.

  ┌─── Capacity = 4 ───┐
  │ 10 │ 20 │ 30 │ __ │  ← size = 3, còn 1 slot trống
  └────────────────────┘

  Push(40):
  │ 10 │ 20 │ 30 │ 40 │  ← size = 4, capacity = 4 (đầy!)

  Push(50):  ← CẦN MỞ RỘNG!
  1. Allocate mảng mới, capacity × 2 = 8
  2. Copy tất cả phần tử cũ sang     ← O(n) operation!
  3. Thêm phần tử mới
  4. Free mảng cũ

  ┌──────── Capacity = 8 ────────────────┐
  │ 10 │ 20 │ 30 │ 40 │ 50 │ __ │ __ │ __│
  └──────────────────────────────────────┘
```

### 1.3 Amortized Analysis — Tại sao push vẫn là O(1)

Mặc dù resize tốn O(n), nhưng nó chỉ xảy ra **rất hiếm** (khi capacity đầy).

```
n lần push, resize doubling strategy:
  Push 1:  copy 0 elements (tạo mới)
  Push 2:  copy 1 element  (resize 1→2)
  Push 3:  copy 2 elements (resize 2→4)
  Push 5:  copy 4 elements (resize 4→8)
  Push 9:  copy 8 elements (resize 8→16)
  ...

  Tổng copies = 1 + 2 + 4 + 8 + ... + n/2 = n - 1
  Tổng operations = n (push) + (n-1) (copy) ≈ 2n
  Amortized cost per push = 2n / n = O(1)
```

> **Interview tip**: Khi interviewer hỏi "push array có thực sự O(1) không?" → trả lời "Amortized O(1) nhờ doubling strategy. Worst case single push là O(n) khi resize, nhưng amortized over n operations thì là O(1)."

### 1.4 Big-O Operations

| Operation                  | Array              | Lý do                    |
| -------------------------- | ------------------ | ------------------------ |
| Access by index            | **O(1)**           | Tính address trực tiếp   |
| Search (unsorted)          | **O(n)**           | Phải duyệt từng phần tử  |
| Search (sorted)            | **O(log n)**       | Binary Search            |
| Insert at end              | **O(1)** amortized | Có thể resize            |
| Insert at beginning/middle | **O(n)**           | Phải dịch chuyển phần tử |
| Delete at end              | **O(1)**           | Chỉ giảm size            |
| Delete at beginning/middle | **O(n)**           | Phải dịch chuyển phần tử |

### 1.6 Patterns phổ biến trong phỏng vấn

**Two Pointers**:

```javascript
// Bài: Tìm 2 số trong sorted array có tổng = target
function twoSum(nums, target) {
  let left = 0,
    right = nums.length - 1;
  while (left < right) {
    const sum = nums[left] + nums[right];
    if (sum === target) return [left, right];
    if (sum < target)
      left++; // cần tổng lớn hơn → tăng left
    else right--; // cần tổng nhỏ hơn → giảm right
  }
  return [-1, -1];
}
// O(n) time, O(1) space — tối ưu!
```

**Sliding Window**:

```javascript
// Bài: Max sum of subarray with size k
function maxSubarraySum(nums, k) {
  let windowSum = 0;
  for (let i = 0; i < k; i++) windowSum += nums[i]; // init window

  let maxSum = windowSum;
  for (let i = k; i < nums.length; i++) {
    windowSum += nums[i] - nums[i - k]; // slide: thêm phải, bỏ trái
    maxSum = Math.max(maxSum, windowSum);
  }
  return maxSum;
}
// O(n) thay vì O(n×k) brute force!
```

**Prefix Sum**:

```javascript
// Bài: Sum of subarray [l, r] in O(1) after O(n) preprocessing
function buildPrefixSum(nums) {
  const prefix = [0]; // prefix[0] = 0 (empty sum)
  for (let i = 0; i < nums.length; i++) {
    prefix.push(prefix[i] + nums[i]);
  }
  return prefix;
}
// Sum(l..r) = prefix[r+1] - prefix[l]
// Query O(1), Build O(n)
```

## 2. Linked List — Danh Sách Liên Kết

### 2.1 Linked List là gì — Từ mức Memory

Khác với Array (contiguous), Linked List lưu phần tử **rải rác** trong memory, kết nối bằng **con trỏ (pointer/reference)**.

```
Singly Linked List:

  Head
   │
   ▼
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ val: 10  │    │ val: 20  │    │ val: 30  │    │ val: 40  │
│ next: ───┼───▶│ next: ───┼───▶│ next: ───┼───▶│ next:null│
│ @0x200   │    │ @0x500   │    │ @0x100   │    │ @0x800   │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
  (rải rác trong memory — không liên tiếp!)

Doubly Linked List:

         Head                                          Tail
          │                                             │
          ▼                                             ▼
     ┌─────────┐     ┌─────────┐     ┌─────────┐  ┌─────────┐
null◀┤prev     │◀────┤prev     │◀────┤prev     │◀─┤prev     │
     │ val: 10 │     │ val: 20 │     │ val: 30 │  │ val: 40 │
     │    next├─────▶│    next├─────▶│    next├──▶│next:null│
     └─────────┘     └─────────┘     └─────────┘  └─────────┘
```

### 2.2 Implementation từ đầu

```javascript
class ListNode {
  constructor(val, next = null) {
    this.val = val;
    this.next = next;
  }
}

class LinkedList {
  constructor() {
    this.head = null;
    this.size = 0;
  }

  // O(1) — Insert at head
  prepend(val) {
    this.head = new ListNode(val, this.head);
    this.size++;
  }

  // O(n) — Insert at tail (O(1) nếu maintain tail pointer)
  append(val) {
    const node = new ListNode(val);
    if (!this.head) {
      this.head = node;
    } else {
      let curr = this.head;
      while (curr.next) curr = curr.next;
      curr.next = node;
    }
    this.size++;
  }

  // O(n) — Delete by value
  delete(val) {
    if (!this.head) return;
    if (this.head.val === val) {
      this.head = this.head.next;
      this.size--;
      return;
    }
    let curr = this.head;
    while (curr.next && curr.next.val !== val) curr = curr.next;
    if (curr.next) {
      curr.next = curr.next.next;
      this.size--;
    }
  }
}
```

### 2.3 Dummy Head (Sentinel Node) — Kỹ thuật quan trọng

```javascript
// Vấn đề: xử lý edge case head riêng biệt rất phiền
// Giải pháp: thêm dummy node trước head

function removeElements(head, val) {
  const dummy = new ListNode(0, head); // dummy → head → ...
  let curr = dummy;
  while (curr.next) {
    if (curr.next.val === val) {
      curr.next = curr.next.next; // skip node
    } else {
      curr = curr.next;
    }
  }
  return dummy.next; // real head
}
// Không cần xử lý đặc biệt khi head bị xóa!
```

### 2.4 Big-O So sánh với Array

| Operation                     | Array          | Linked List              | Thắng?      |
| ----------------------------- | -------------- | ------------------------ | ----------- |
| Access [i]                    | **O(1)**       | O(n)                     | Array       |
| Search                        | O(n)           | O(n)                     | Hòa         |
| Insert at head                | O(n)           | **O(1)**                 | Linked List |
| Insert at tail                | O(1) amortized | O(n) / **O(1)** với tail | Tùy         |
| Insert at middle (given node) | O(n)           | **O(1)**                 | Linked List |
| Delete at head                | O(n)           | **O(1)**                 | Linked List |
| Cache performance             | **Excellent**  | Poor (scattered memory)  | Array       |
| Memory overhead               | Chỉ data       | data + pointer(s)        | Array       |

> **Khi nào dùng Linked List?**: Khi cần **insert/delete thường xuyên ở đầu/giữa** và **không cần random access**. Ví dụ: LRU Cache (doubly linked list + hash map), undo history, task schedulers.

### 2.5 Patterns phỏng vấn kinh điển

**Reverse Linked List** (LC #206 — bắt buộc!):

```javascript
function reverseList(head) {
  let prev = null,
    curr = head;
  while (curr) {
    const next = curr.next; // save
    curr.next = prev; // reverse
    prev = curr; // advance
    curr = next;
  }
  return prev;
}
// prev → curr → next    →    prev ← curr    next
//                             (reverse arrow)
```

**Fast & Slow Pointers (Floyd's)** — phát hiện cycle:

```javascript
function hasCycle(head) {
  let slow = head,
    fast = head;
  while (fast && fast.next) {
    slow = slow.next; // 1 bước
    fast = fast.next.next; // 2 bước
    if (slow === fast) return true; // gặp nhau → có cycle!
  }
  return false; // fast đến null → không có cycle
}

// Tại sao hoạt động?
// Nếu có cycle, fast "đuổi" slow với tốc độ gấp đôi
// Khoảng cách giảm 1 mỗi bước → chắc chắn gặp nhau
// Giống 2 người chạy trên sân cỏ vòng tròn
```

**Tìm middle node**:

```javascript
function middleNode(head) {
  let slow = head,
    fast = head;
  while (fast && fast.next) {
    slow = slow.next;
    fast = fast.next.next;
  }
  return slow; // slow đứng ở giữa khi fast đến cuối
}
```

## 3. Stack — Ngăn Xếp

### 3.1 Stack là gì

**LIFO** — Last In, First Out. Phần tử vào sau lấy ra trước.

```
Push(10)  Push(20)  Push(30)  Pop()→30  Pop()→20

│      │  │      │  │  30  │  │      │  │      │
│      │  │  20  │  │  20  │  │  20  │  │      │
│  10  │  │  10  │  │  10  │  │  10  │  │  10  │
└──────┘  └──────┘  └──────┘  └──────┘  └──────┘

Tất cả operations: O(1)
  push(val) — thêm lên đỉnh
  pop()     — lấy và xóa đỉnh
  peek()    — xem đỉnh (không xóa)
  isEmpty() — kiểm tra rỗng
```

### 3.2 Stack ở đâu trong thực tế?

```
1. CALL STACK — Function execution (cực quan trọng!)

   function a() { b(); }      Call Stack:
   function b() { c(); }      ┌───────┐
   function c() { ... }       │  c()  │ ← đang chạy
   a();                       │  b()  │
                               │  a()  │
                               │ main  │
                               └───────┘
   Stack Overflow = call stack quá sâu (recursion vô hạn)

2. UNDO/REDO — Text editors
   Mỗi action push vào undo stack
   Undo: pop undo stack, push vào redo stack

3. BROWSER HISTORY — Back/Forward buttons

4. EXPRESSION EVALUATION — Compilers, calculators
```

### 3.3 Implementation

```javascript
// Cách 1: Dùng Array (đơn giản, thường dùng trong phỏng vấn)
class Stack {
  constructor() {
    this.items = [];
  }
  push(val) {
    this.items.push(val);
  }
  pop() {
    return this.items.pop();
  }
  peek() {
    return this.items[this.items.length - 1];
  }
  isEmpty() {
    return this.items.length === 0;
  }
  size() {
    return this.items.length;
  }
}

// Cách 2: Dùng Linked List (O(1) guaranteed, no amortization)
class StackNode {
  constructor(val, next = null) {
    this.val = val;
    this.next = next;
  }
}
class LinkedStack {
  constructor() {
    this.top = null;
    this._size = 0;
  }
  push(val) {
    this.top = new StackNode(val, this.top);
    this._size++;
  }
  pop() {
    if (!this.top) return undefined;
    const val = this.top.val;
    this.top = this.top.next;
    this._size--;
    return val;
  }
  peek() {
    return this.top?.val;
  }
  isEmpty() {
    return !this.top;
  }
}
```

### 3.4 Patterns phỏng vấn

**Valid Parentheses (LC #20)**:

```javascript
function isValid(s) {
  const stack = [];
  const map = { ")": "(", "]": "[", "}": "{" };
  for (const c of s) {
    if (c in map) {
      if (stack.pop() !== map[c]) return false; // không match
    } else {
      stack.push(c); // opening bracket
    }
  }
  return stack.length === 0; // tất cả đã match
}
```

**Monotonic Stack — Next Greater Element (LC #496)**:

```javascript
// Tìm phần tử lớn hơn tiếp theo cho mỗi element
function nextGreaterElement(nums) {
  const result = new Array(nums.length).fill(-1);
  const stack = []; // stack chứa INDEX, giảm dần

  for (let i = 0; i < nums.length; i++) {
    // Pop tất cả elements nhỏ hơn nums[i]
    while (stack.length && nums[stack[stack.length - 1]] < nums[i]) {
      result[stack.pop()] = nums[i]; // nums[i] là next greater
    }
    stack.push(i);
  }
  return result;
}
// Input:  [2, 1, 2, 4, 3]
// Output: [4, 2, 4, -1, -1]
// O(n) — mỗi element push/pop tối đa 1 lần!
```

**Min Stack — O(1) getMin (LC #155)**:

```javascript
class MinStack {
  constructor() {
    this.stack = [];
    this.minStack = []; // track min tại mỗi thời điểm
  }
  push(val) {
    this.stack.push(val);
    const curMin = this.minStack.length
      ? Math.min(val, this.minStack[this.minStack.length - 1])
      : val;
    this.minStack.push(curMin);
  }
  pop() {
    this.stack.pop();
    this.minStack.pop();
  }
  getMin() {
    return this.minStack[this.minStack.length - 1];
  }
}
// Trick: dùng 2 stacks — 1 cho data, 1 track min song song
```

## 4. Queue — Hàng Đợi

### 4.1 Queue là gì

**FIFO** — First In, First Out. Phần tử vào trước lấy ra trước. Giống xếp hàng mua đồ.

```
Enqueue: thêm vào cuối (rear)        Dequeue: lấy từ đầu (front)

  Enqueue(10)  Enqueue(20)  Enqueue(30)  Dequeue()→10

  Front→│ 10 │  │ 10 │ 20 │  │ 10 │ 20 │ 30 │  │ 20 │ 30 │
        └────┘  └────┴────┘  └────┴────┴────┘  └────┴────┘
                                                  Front↑
```

### 4.2 Implementations

```javascript
// ❌ Array-based (CHẬM! — dequeue = shift() = O(n))
const queue = [];
queue.push(1); // enqueue O(1)
queue.shift(); // dequeue O(n)! — phải dịch tất cả phần tử

// ✅ Linked List-based (O(1) for both)
class Queue {
  constructor() {
    this.head = null; // front
    this.tail = null; // rear
    this._size = 0;
  }
  enqueue(val) {
    const node = { val, next: null };
    if (this.tail) this.tail.next = node;
    else this.head = node;
    this.tail = node;
    this._size++;
  }
  dequeue() {
    if (!this.head) return undefined;
    const val = this.head.val;
    this.head = this.head.next;
    if (!this.head) this.tail = null;
    this._size--;
    return val;
  }
  peek() {
    return this.head?.val;
  }
  isEmpty() {
    return !this.head;
  }
  size() {
    return this._size;
  }
}
```

### 4.3 Deque (Double-Ended Queue)

Thêm/xóa ở **cả hai đầu** trong O(1). Kết hợp Stack + Queue.

```
        Front                    Rear
          │                       │
          ▼                       ▼
     ┌─────────────────────────────┐
     │  10  │  20  │  30  │  40   │
     └─────────────────────────────┘
     ◄── addFront / removeFront    addRear / removeRear ──►

Use cases:
  - Sliding Window Maximum (LC #239) — classic!
  - BFS with 0-1 edge weights
  - Work stealing in thread pools
```

### 4.4 Circular Queue

```javascript
// Array-based circular queue — hiệu quả hơn shift()
class CircularQueue {
  constructor(capacity) {
    this.items = new Array(capacity);
    this.capacity = capacity;
    this.front = 0;
    this.rear = -1;
    this._size = 0;
  }
  enqueue(val) {
    if (this._size === this.capacity) return false; // full
    this.rear = (this.rear + 1) % this.capacity; // wrap around!
    this.items[this.rear] = val;
    this._size++;
    return true;
  }
  dequeue() {
    if (this._size === 0) return undefined;
    const val = this.items[this.front];
    this.front = (this.front + 1) % this.capacity; // wrap around!
    this._size--;
    return val;
  }
}
// Không cần shift! → O(1) dequeue
// Modulo trick: (index + 1) % capacity → quay vòng khi đến cuối
```

### 4.5 Queue ở đâu trong thực tế?

```
Backend / System Design:
  - Message Queues (RabbitMQ, SQS, Kafka) — core concept
  - Task queues (BullMQ in Node.js)
  - Rate limiting (request queues)
  - Print Queue, CPU scheduling (Ready Queue)
  - Event Loop callback queue!

Algorithms:
  - BFS (Breadth-First Search) — luôn dùng queue
  - Level-order tree traversal
  - Topological sort (Kahn's algorithm)
```

**BFS Template**:

```javascript
function bfs(graph, start) {
  const queue = [start];
  const visited = new Set([start]);

  while (queue.length > 0) {
    const node = queue.shift(); // dequeue (dùng LinkedList queue cho O(1))
    // process(node)

    for (const neighbor of graph[node]) {
      if (!visited.has(neighbor)) {
        visited.add(neighbor);
        queue.push(neighbor); // enqueue
      }
    }
  }
}
```

### 5.2 Câu hỏi phỏng vấn

**Q: Array vs Linked List — khi nào dùng cái nào?**

> A: Array khi cần random access nhanh O(1), data size dự đoán được, và cache performance quan trọng. Linked List khi cần insert/delete O(1) ở đầu/giữa thường xuyên, size thay đổi liên tục. Trong thực tế, array chiếm đa số use cases nhờ cache-friendliness. Linked List chủ yếu dùng trong LRU Cache, OS task scheduling, và memory allocators.

**Q: Tại sao JS `Array.shift()` là O(n)?**

> A: `shift()` xóa phần tử đầu, phải dịch tất cả phần tử còn lại sang trái 1 vị trí. Cho n phần tử → n-1 lần di chuyển → O(n). Giải pháp: dùng linked list-based queue, hoặc circular buffer, hoặc đơn giản là dùng index pointer thay vì shift.

**Q: Implement stack bằng 2 queues? Queue bằng 2 stacks?**

> A: Classic interview question — cần nắm vững cả hai hướng. Queue bằng 2 stacks: push stack (inbox) + pop stack (outbox). Khi pop: nếu outbox trống, đổ hết inbox sang outbox. Amortized O(1) per operation.
