## 1. DP Là Gì — Tư Duy Cốt Lõi

DP giải bài toán bằng cách **chia thành các bài toán con (subproblems)** và **lưu trữ kết quả** (cache) để tránh tính lại.

### 2 điều kiện để dùng DP:

```
1. OPTIMAL SUBSTRUCTURE:
   Giải pháp tối ưu của bài toán CÓ THỂ xây dựng từ giải pháp tối ưu
   của các bài toán con.

   Ví dụ: Shortest path A→D qua B,C
   = shortest(A→B) + shortest(B→D)   ← optimal substructure!

2. OVERLAPPING SUBPROBLEMS:
   Các bài toán con LẶP LẠI nhiều lần.

   Fibonacci: fib(5)
                    fib(5)
                  /        \
              fib(4)       fib(3)    ← fib(3) tính 2 lần!
             /     \       /    \
         fib(3)  fib(2) fib(2) fib(1) ← fib(2) tính 3 lần!
        /    \
    fib(2) fib(1)

   Không có DP: O(2^n) — tính lại rất nhiều
   Có DP (lưu cache): O(n) — mỗi subproblem chỉ tính 1 lần!
```

### Framework giải bài DP:

```
Bước 1: Define state — dp[i] hoặc dp[i][j] đại diện cho gì?
Bước 2: Recurrence relation — dp[i] phụ thuộc dp[?] thế nào?
Bước 3: Base cases — dp[0] = ?, dp[1] = ?
Bước 4: Order of computation — fill dp table theo thứ tự nào?
Bước 5: Answer location — kết quả ở dp[n] hay max(dp)?
```

## 2. Top-Down vs Bottom-Up

### Top-Down (Memoization) — Recursive + Cache

```javascript
// Fibonacci — Top-Down
function fib(n, memo = {}) {
  if (n <= 1) return n;
  if (memo[n] !== undefined) return memo[n]; // đã tính rồi → return
  memo[n] = fib(n - 1, memo) + fib(n - 2, memo);
  return memo[n];
}
// Bắt đầu từ bài toán LỚN (n), chia xuống bài toán NHỎ (n-1, n-2)
```

### Bottom-Up (Tabulation) — Iterative + DP Table

```javascript
// Fibonacci — Bottom-Up
function fib(n) {
  if (n <= 1) return n;
  const dp = new Array(n + 1);
  dp[0] = 0;
  dp[1] = 1; // base cases
  for (let i = 2; i <= n; i++) {
    dp[i] = dp[i - 1] + dp[i - 2]; // fill table từ NHỎ đến LỚN
  }
  return dp[n];
}
// Không recursion → không stack overflow risk
// Dễ optimize space (xem section 6)
```

|             | Top-Down                         | Bottom-Up               |
| ----------- | -------------------------------- | ----------------------- |
| Approach    | Recursive + memo                 | Iterative + table       |
| Code        | Trực giác hơn                    | Cần suy nghĩ order      |
| Stack       | Recursion stack (risk overflow)  | ✅ No stack risk        |
| Performance | Slightly slower (function calls) | ✅ Slightly faster      |
| Subproblems | Chỉ tính subproblems CẦN thiết   | Tính TẤT CẢ subproblems |
| Interview   | Dễ code nhanh                    | Dễ optimize space       |

## 3. 1D DP Patterns

### Climbing Stairs (LC #70)

```javascript
// State: dp[i] = số cách leo đến bậc i
// Recurrence: dp[i] = dp[i-1] + dp[i-2]  (từ bậc i-1 lên 1, hoặc i-2 lên 2)
// Base: dp[0] = 1, dp[1] = 1

function climbStairs(n) {
  let prev2 = 1,
    prev1 = 1;
  for (let i = 2; i <= n; i++) {
    const curr = prev1 + prev2;
    prev2 = prev1;
    prev1 = curr;
  }
  return prev1;
}
// Space optimized: O(1) thay vì O(n)
```

### House Robber (LC #198)

```javascript
// State: dp[i] = max money cướp được từ house 0..i
// Recurrence: dp[i] = max(dp[i-1], dp[i-2] + nums[i])
//   Hoặc KHÔNG cướp house i (dp[i-1]), hoặc CÓ cướp house i (dp[i-2] + nums[i])
// Base: dp[0] = nums[0], dp[1] = max(nums[0], nums[1])

function rob(nums) {
  if (nums.length === 1) return nums[0];
  let prev2 = nums[0],
    prev1 = Math.max(nums[0], nums[1]);
  for (let i = 2; i < nums.length; i++) {
    const curr = Math.max(prev1, prev2 + nums[i]);
    prev2 = prev1;
    prev1 = curr;
  }
  return prev1;
}
```

### Longest Increasing Subsequence (LC #300)

```javascript
// State: dp[i] = LIS length ending at index i
// Recurrence: dp[i] = max(dp[j] + 1) for all j < i where nums[j] < nums[i]
// Base: dp[i] = 1 (mỗi element tự nó là LIS length 1)

function lengthOfLIS(nums) {
  const dp = new Array(nums.length).fill(1);
  let maxLen = 1;
  for (let i = 1; i < nums.length; i++) {
    for (let j = 0; j < i; j++) {
      if (nums[j] < nums[i]) dp[i] = Math.max(dp[i], dp[j] + 1);
    }
    maxLen = Math.max(maxLen, dp[i]);
  }
  return maxLen;
}
// O(n²) — có thể O(n log n) với patience sorting + binary search
```

## 4. 2D DP Patterns

### Unique Paths (LC #62)

```javascript
// State: dp[i][j] = số cách đi từ (0,0) đến (i,j)
// Recurrence: dp[i][j] = dp[i-1][j] + dp[i][j-1] (từ trên xuống + từ trái sang)
// Base: dp[0][j] = 1, dp[i][0] = 1 (chỉ 1 cách đi thẳng)

function uniquePaths(m, n) {
  const dp = Array.from({ length: m }, () => new Array(n).fill(1));
  for (let i = 1; i < m; i++) {
    for (let j = 1; j < n; j++) {
      dp[i][j] = dp[i - 1][j] + dp[i][j - 1];
    }
  }
  return dp[m - 1][n - 1];
}
```

### Longest Common Subsequence (LC #1143)

```javascript
// State: dp[i][j] = LCS length of text1[0..i-1] và text2[0..j-1]
// Recurrence:
//   text1[i-1] === text2[j-1] → dp[i][j] = dp[i-1][j-1] + 1
//   Else → dp[i][j] = max(dp[i-1][j], dp[i][j-1])

function longestCommonSubsequence(text1, text2) {
  const m = text1.length,
    n = text2.length;
  const dp = Array.from({ length: m + 1 }, () => new Array(n + 1).fill(0));
  for (let i = 1; i <= m; i++) {
    for (let j = 1; j <= n; j++) {
      if (text1[i - 1] === text2[j - 1]) dp[i][j] = dp[i - 1][j - 1] + 1;
      else dp[i][j] = Math.max(dp[i - 1][j], dp[i][j - 1]);
    }
  }
  return dp[m][n];
}
```

### 0/1 Knapsack

```javascript
// State: dp[i][w] = max value dùng items[0..i-1] với capacity w
// Recurrence: dp[i][w] = max(dp[i-1][w], dp[i-1][w-wt[i]] + val[i])
//   Không chọn item i: dp[i-1][w]
//   Chọn item i (nếu đủ capacity): dp[i-1][w-wt[i]] + val[i]

function knapsack(weights, values, W) {
  const n = weights.length;
  const dp = Array.from({ length: n + 1 }, () => new Array(W + 1).fill(0));
  for (let i = 1; i <= n; i++) {
    for (let w = 0; w <= W; w++) {
      dp[i][w] = dp[i - 1][w]; // không chọn
      if (weights[i - 1] <= w) {
        dp[i][w] = Math.max(
          dp[i][w],
          dp[i - 1][w - weights[i - 1]] + values[i - 1],
        );
      }
    }
  }
  return dp[n][W];
}
```

## 5. Classic DP Problems

| Problem                       | Pattern                    | Key Insight                       |
| ----------------------------- | -------------------------- | --------------------------------- |
| LC #70 Climbing Stairs        | 1D linear                  | dp[i] = dp[i-1] + dp[i-2]         |
| LC #198 House Robber          | 1D take/skip               | max(take + dp[i-2], skip dp[i-1]) |
| LC #300 LIS                   | 1D quadratic               | dp[i] = max(dp[j]+1) for j < i    |
| LC #322 Coin Change           | Unbounded knapsack         | dp[i] = min(dp[i-coin]+1)         |
| LC #62 Unique Paths           | 2D grid                    | dp[i][j] = up + left              |
| LC #1143 LCS                  | 2D string match            | match → diagonal+1, else max      |
| LC #416 Partition Equal Sum   | 0/1 knapsack variant       | dp[j] = can reach sum j?          |
| LC #518 Coin Change 2         | Unbounded knapsack (count) | dp[j] += dp[j-coin]               |
| LC #72 Edit Distance          | 2D string                  | insert/delete/replace             |
| LC #139 Word Break            | 1D substring               | dp[i] = any dp[j] && word exists? |
| LC #152 Max Product Subarray  | 1D track min/max           | Negative × negative = positive    |
| LC #5 Longest Palindromic Sub | 2D expand or dp            | dp[i][j] = dp[i+1][j-1]           |

## 6. Space Optimization

```javascript
// 2D → 1D khi dp[i] chỉ phụ thuộc dp[i-1]

// Knapsack space optimized: O(W) thay vì O(n×W)
function knapsackOptimized(weights, values, W) {
  const dp = new Array(W + 1).fill(0);
  for (let i = 0; i < weights.length; i++) {
    for (let w = W; w >= weights[i]; w--) {
      // ĐI NGƯỢC để tránh dùng lại item
      dp[w] = Math.max(dp[w], dp[w - weights[i]] + values[i]);
    }
  }
  return dp[W];
}

// Coin Change: đi XUÔI (vì unbounded — dùng lại coin được)
function coinChange(coins, amount) {
  const dp = new Array(amount + 1).fill(Infinity);
  dp[0] = 0;
  for (const coin of coins) {
    for (let j = coin; j <= amount; j++) {
      // ĐI XUÔI
      dp[j] = Math.min(dp[j], dp[j - coin] + 1);
    }
  }
  return dp[amount] === Infinity ? -1 : dp[amount];
}
```

## 7. Câu Hỏi Phỏng Vấn

**Q: DP vs Greedy — khi nào dùng cái nào?**

> A: **Greedy**: bài có "greedy choice property" — chọn tối ưu local → tối ưu global (ví dụ: activity selection, Huffman). **DP**: khi greedy KHÔNG đúng — cần thử mọi khả năng nhưng các subproblems overlap. Rule of thumb: nếu bài hỏi "minimum/maximum/count ways" và brute force là exponential → thử DP.

**Q: Nhận diện bài DP thế nào?**

> A: (1) Hỏi optimization (min/max) hoặc count (how many ways). (2) Có choices tại mỗi step. (3) Brute force recursive → thấy overlapping subproblems. (4) Keywords: "subsequence", "substring", "partition", "path", "target sum".

**Q: 0/1 Knapsack vs Unbounded Knapsack khác gì?**

> A: **0/1**: mỗi item chỉ dùng 1 lần → space optimized loop ĐI NGƯỢC (w → 0). **Unbounded**: item dùng nhiều lần → loop ĐI XUÔI (0 → w). Coin Change = unbounded knapsack.
