## 1. Binary Tree — Cơ Bản

### 1.1 Cấu trúc

```
Mỗi node có tối đa 2 children: left và right.

           1             ← root
          / \
         2   3           ← children of 1
        / \   \
       4   5   6         ← leaves (lá): nodes không có children
      /
     7

Properties:
  - Depth (độ sâu): khoảng cách từ root. root = depth 0. Node 4 = depth 2.
  - Height (chiều cao): path dài nhất từ node đến leaf. root = height 3.
  - Total nodes: tối đa 2^(h+1) - 1 cho tree height h.
```

### 1.2 Các loại Binary Tree

```
Full Binary Tree:               Complete Binary Tree:
Mỗi node có 0 hoặc 2 children   Tất cả levels đầy, trừ level cuối
                                 (level cuối fill từ trái sang)
       1                                1
      / \                              / \
     2   3                            2   3
    / \                              / \ /
   4   5                            4  5 6

Perfect Binary Tree:             Balanced Binary Tree:
Tất cả levels đầy                |height(left) - height(right)| ≤ 1
       1                         cho MỌI node
      / \
     2   3                              1
    / \ / \                            / \
   4  5 6  7                          2   3
                                     / \   \
                                    4   5   6
```

### 1.3 Node Implementation

```javascript
class TreeNode {
  constructor(val, left = null, right = null) {
    this.val = val;
    this.left = left;
    this.right = right;
  }
}

// Tạo tree:
//     1
//    / \
//   2   3
const root = new TreeNode(1, new TreeNode(2), new TreeNode(3));
```

## 2. Tree Traversals — Duyệt Cây

### 2.1 Bốn cách duyệt

```
Cho tree:       1
               / \
              2   3
             / \
            4   5

Inorder   (Left → Root → Right):  4, 2, 5, 1, 3   ← BST: sorted order!
Preorder  (Root → Left → Right):  1, 2, 4, 5, 3   ← Dùng cho serialize
Postorder (Left → Right → Root):  4, 5, 2, 3, 1   ← Dùng cho delete
Level-order (BFS):                1, 2, 3, 4, 5    ← Theo từng tầng
```

### 2.2 Recursive Implementation

```javascript
// Inorder (quan trọng nhất cho BST)
function inorder(root, result = []) {
  if (!root) return result;
  inorder(root.left, result); // L
  result.push(root.val); // Root
  inorder(root.right, result); // R
  return result;
}

// Preorder
function preorder(root, result = []) {
  if (!root) return result;
  result.push(root.val); // Root
  preorder(root.left, result); // L
  preorder(root.right, result); // R
  return result;
}

// Postorder
function postorder(root, result = []) {
  if (!root) return result;
  postorder(root.left, result); // L
  postorder(root.right, result); // R
  result.push(root.val); // Root
  return result;
}
```

### 2.3 Iterative Inorder (dùng Stack)

```javascript
function inorderIterative(root) {
  const result = [],
    stack = [];
  let curr = root;
  while (curr || stack.length) {
    while (curr) {
      // đi sâu bên trái
      stack.push(curr);
      curr = curr.left;
    }
    curr = stack.pop(); // backtrack
    result.push(curr.val); // visit
    curr = curr.right; // chuyển sang phải
  }
  return result;
}
```

### 2.4 Level-Order (BFS dùng Queue)

```javascript
function levelOrder(root) {
  if (!root) return [];
  const result = [],
    queue = [root];
  while (queue.length) {
    const level = [];
    const size = queue.length; // fix size trước khi loop!
    for (let i = 0; i < size; i++) {
      const node = queue.shift();
      level.push(node.val);
      if (node.left) queue.push(node.left);
      if (node.right) queue.push(node.right);
    }
    result.push(level);
  }
  return result;
}
// [[1], [2,3], [4,5]] — mỗi inner array = 1 level
```

## 3. Binary Search Tree (BST)

### 3.1 BST Property

**Với mọi node**: tất cả nodes bên trái **< node** < tất cả nodes bên phải.

```
BST hợp lệ:          KHÔNG hợp lệ:
       8                    8
      / \                  / \
     3   10               3   10
    / \    \             / \    \
   1   6   14           1   6   14
      / \                  / \
     4   7                4   12  ← 12 > 8, phải ở phải 8!
```

### 3.2 BST Operations

```javascript
// SEARCH — O(log n) average, O(n) worst (skewed)
function search(root, target) {
  if (!root) return null;
  if (target === root.val) return root;
  if (target < root.val) return search(root.left, target);
  return search(root.right, target);
}

// INSERT — O(log n) average
function insert(root, val) {
  if (!root) return new TreeNode(val);
  if (val < root.val) root.left = insert(root.left, val);
  else if (val > root.val) root.right = insert(root.right, val);
  return root; // duplicate → ignore
}

// DELETE — O(log n) average — PHỨC TẠP NHẤT
function deleteNode(root, key) {
  if (!root) return null;
  if (key < root.val) {
    root.left = deleteNode(root.left, key);
  } else if (key > root.val) {
    root.right = deleteNode(root.right, key);
  } else {
    // Tìm thấy node cần xóa
    // Case 1 & 2: 0 hoặc 1 child
    if (!root.left) return root.right;
    if (!root.right) return root.left;
    // Case 3: 2 children → thay bằng inorder successor (nhỏ nhất bên phải)
    let successor = root.right;
    while (successor.left) successor = successor.left;
    root.val = successor.val;
    root.right = deleteNode(root.right, successor.val);
  }
  return root;
}
```

### 3.3 BST Big-O

| Operation | Average  | Worst (Skewed) |
| --------- | -------- | -------------- |
| Search    | O(log n) | O(n)           |
| Insert    | O(log n) | O(n)           |
| Delete    | O(log n) | O(n)           |
| Inorder   | O(n)     | O(n)           |

> **Worst case O(n)** xảy ra khi BST bị **skewed** (lệch) — insert 1,2,3,4,5 tuần tự → thành linked list! Giải pháp: **self-balancing trees** (AVL, Red-Black).

## 4. Self-Balancing Trees — AVL & Red-Black

### 4.1 AVL Tree

Duy trì |height(left) - height(right)| ≤ 1 cho **mọi node** bằng rotations.

```
4 loại rotation:

Right Rotation (LL case):       Left Rotation (RR case):
      z                   y           z                 y
     / \                / \          / \               / \
    y   T4   →         x   z       T1  y     →       z   x
   / \                / \ / \          / \           / \ / \
  x   T3             T1 T2 T3 T4     T2  x         T1 T2 T3 T4
 / \                                     / \
T1  T2                                  T3 T4

Left-Right (LR):                Right-Left (RL):
First left rotate on y,         First right rotate on y,
then right rotate on z          then left rotate on z
```

### 4.2 Red-Black Tree

Dùng "màu" (Red/Black) + 5 rules để đảm bảo balanced. **Less strict** hơn AVL — cho phép height chênh lệch gấp đôi.

**Quy tắc**: (1) Mỗi node là Red hoặc Black, (2) Root luôn Black, (3) Leaves (NIL) luôn Black, (4) Red node → children phải Black (no 2 consecutive reds), (5) Mọi path root→leaf có **cùng số Black nodes**.

|                | AVL                         | Red-Black                           |
| -------------- | --------------------------- | ----------------------------------- |
| Balance factor | ≤ 1                         | height ≤ 2 × log(n+1)               |
| Search         | Nhanh hơn (strict balanced) | Chậm hơn một chút                   |
| Insert/Delete  | Chậm hơn (nhiều rotations)  | Nhanh hơn (ít rotations)            |
| Dùng bởi       | Database indexes            | `std::map`, Java TreeMap, Linux CFS |

> **Interview level**: Biết tại sao cần self-balancing, hiểu concept rotation. KHÔNG cần implement AVL/RB tree from scratch (trừ Google).

## 5. Heap — Đống

### 5.1 Heap là gì

**Complete Binary Tree** thỏa mãn heap property:

- **Min-Heap**: parent ≤ children (root = minimum)
- **Max-Heap**: parent ≥ children (root = maximum)

```
Min-Heap:            Max-Heap:
       1                   9
      / \                 / \
     3   5               7   8
    / \ / \             / \ /
   7  8 9  6           4  5 6

Array representation (vì complete → contiguous):
Min-Heap: [1, 3, 5, 7, 8, 9, 6]
           0  1  2  3  4  5  6

Parent of i:       Math.floor((i - 1) / 2)
Left child of i:   2 * i + 1
Right child of i:  2 * i + 2
```

### 5.2 Heap Operations

```javascript
class MinHeap {
  constructor() {
    this.data = [];
  }

  // O(log n) — thêm vào cuối, "bubble up"
  insert(val) {
    this.data.push(val);
    this._bubbleUp(this.data.length - 1);
  }

  // O(log n) — lấy root, đặt phần tử cuối lên root, "sink down"
  extractMin() {
    if (this.data.length === 0) return undefined;
    const min = this.data[0];
    const last = this.data.pop();
    if (this.data.length > 0) {
      this.data[0] = last;
      this._sinkDown(0);
    }
    return min;
  }

  peek() {
    return this.data[0];
  } // O(1)

  _bubbleUp(i) {
    while (i > 0) {
      const parent = Math.floor((i - 1) / 2);
      if (this.data[i] >= this.data[parent]) break;
      [this.data[i], this.data[parent]] = [this.data[parent], this.data[i]];
      i = parent;
    }
  }

  _sinkDown(i) {
    const n = this.data.length;
    while (true) {
      let smallest = i;
      const left = 2 * i + 1,
        right = 2 * i + 2;
      if (left < n && this.data[left] < this.data[smallest]) smallest = left;
      if (right < n && this.data[right] < this.data[smallest]) smallest = right;
      if (smallest === i) break;
      [this.data[i], this.data[smallest]] = [this.data[smallest], this.data[i]];
      i = smallest;
    }
  }
}
```

### 5.3 Heapify — Build Heap O(n)

```javascript
// Build heap từ array — O(n) (KHÔNG phải O(n log n)!)
function buildMinHeap(arr) {
  // Bắt đầu từ node cuối có children, sinkDown
  for (let i = Math.floor(arr.length / 2) - 1; i >= 0; i--) {
    sinkDown(arr, i, arr.length);
  }
}
// Tại sao O(n)? Phần lớn nodes ở tầng dưới → ít work (sinkDown ngắn)
// Tổng work = Σ h_i = O(n)  (chứng minh bằng series)
```

### 5.4 Khi nào dùng Heap

| Use case              | Tại sao Heap                                  |
| --------------------- | --------------------------------------------- |
| Top K elements        | Maintain heap size K → O(n log k)             |
| Priority Queue        | Extract min/max O(log n)                      |
| Merge K sorted lists  | Min-heap of K elements                        |
| Median of data stream | Max-heap (lower half) + Min-heap (upper half) |
| Dijkstra's algorithm  | Priority queue for shortest path              |
| Task scheduling (OS)  | Priority-based scheduling                     |

## 6. Trie — Prefix Tree

```
Lưu tập hợp strings, tối ưu cho prefix search.

Insert: "cat", "car", "card", "dog"

         root
        /    \
       c      d
       |      |
       a      o
      / \     |
     t   r    g*
     *   |
         d*

  * = end of word (isEnd = true)

Search("car"):  root→c→a→r → isEnd? YES ✓
Search("ca"):   root→c→a → isEnd? NO (prefix, not complete word)
StartsWith("ca"): root→c→a → EXISTS ✓
```

```javascript
class TrieNode {
  constructor() {
    this.children = {}; // or Map() or Array(26)
    this.isEnd = false;
  }
}

class Trie {
  constructor() {
    this.root = new TrieNode();
  }

  insert(word) {
    let node = this.root;
    for (const c of word) {
      if (!node.children[c]) node.children[c] = new TrieNode();
      node = node.children[c];
    }
    node.isEnd = true;
  }

  search(word) {
    const node = this._traverse(word);
    return node !== null && node.isEnd;
  }

  startsWith(prefix) {
    return this._traverse(prefix) !== null;
  }

  _traverse(str) {
    let node = this.root;
    for (const c of str) {
      if (!node.children[c]) return null;
      node = node.children[c];
    }
    return node;
  }
}
// Insert/Search: O(m) — m = word length
// Space: O(TOTAL_CHARACTERS)
```

**Use cases**: Autocomplete, spell checker, IP routing (longest prefix match), contacts search.

## 8. Câu Hỏi Phỏng Vấn

**Q: BST worst case O(n) — làm sao khắc phục?**

> A: Dùng self-balancing BST (AVL hoặc Red-Black Tree). AVL maintain |height difference| ≤ 1 bằng rotations sau mỗi insert/delete → guarantee O(log n). Red-Black tree ít strict hơn nhưng ít rotations hơn → insert/delete nhanh hơn AVL. Databases dùng B-Tree (multi-way balanced) cho disk-friendly indexing.

**Q: Heap vs BST — khi nào dùng cái nào?**

> A: **Heap**: khi chỉ cần extract min/max nhanh (Priority Queue). Insert/Extract O(log n), nhưng search arbitrary element O(n). **BST**: khi cần search, insert, delete arbitrary elements đều O(log n), và cần sorted order (inorder traversal). Heap tốt hơn cho "top K" problems; BST tốt hơn cho range queries.

**Q: Build heap tại sao O(n) mà không phải O(n log n)?**

> A: Nếu insert từng element → O(n log n). Nhưng `heapify` bottom-up: bắt đầu từ nodes gần leaf (tầng dưới) → ít work per node. Tầng dưới cùng: n/2 nodes, sinkDown 0 bước. Tầng trên: n/4 nodes, 1 bước. Tổng = n/4 × 1 + n/8 × 2 + n/16 × 3 + ... ≈ n. (Taylor series sum).
