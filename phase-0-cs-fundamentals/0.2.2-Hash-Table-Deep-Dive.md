## 1. Hash Table Là Gì

Hash Table (hay Hash Map) lưu trữ dữ liệu dạng **key-value pairs** với khả năng **truy cập O(1) average** cho insert, search, delete.

```
Ý tưởng cốt lõi:

  key ───▶ hash(key) ───▶ index ───▶ value

  Ví dụ: lưu trữ phonebook

  hash("Alice") = 2        Bucket Array:
  hash("Bob")   = 5        ┌─────┬──────────────────────┐
  hash("Charlie")= 2       │  0  │ (empty)              │
                            │  1  │ (empty)              │
  "Alice" và "Charlie"     │  2  │ Alice→555 → Charlie→777│ ← COLLISION!
  cùng hash = 2            │  3  │ (empty)              │
  → COLLISION! → cần giải  │  4  │ (empty)              │
    quyết                  │  5  │ Bob→666              │
                            │  6  │ (empty)              │
                            └─────┴──────────────────────┘
```

## 2. Hash Functions

### 2.1 Hash function tốt cần gì?

1. **Deterministic**: Cùng input → cùng output (luôn luôn)
2. **Uniform distribution**: Phân bố đều các keys vào buckets → ít collision
3. **Fast**: Tính nhanh, ideally O(1) hoặc O(key_length)

### 2.2 Các kỹ thuật hash phổ biến

**Division method**: `hash(key) = key % table_size`

```
hash(15) = 15 % 7 = 1
hash(22) = 22 % 7 = 1  ← collision!
// table_size nên là số nguyên tố → phân bố đều hơn
```

**Multiplication method**: `hash(key) = floor(table_size × (key × A mod 1))`

- A ≈ 0.6180339887 (golden ratio -1) → Knuth recommendation

**String hashing**:

```javascript
// Polynomial rolling hash — phổ biến cho strings
function hash(str, tableSize) {
  const PRIME = 31;
  let hashValue = 0;
  for (let i = 0; i < str.length; i++) {
    hashValue = (hashValue * PRIME + str.charCodeAt(i)) % tableSize;
  }
  return hashValue;
}
// "abc": hash = (0*31 + 97)*31 + 98)*31 + 99 = 96354
// Dùng prime (31) để giảm collision pattern
```

## 3. Collision Resolution

### 3.1 Separate Chaining (Dùng Linked List)

```
Mỗi bucket là 1 linked list. Collision → append vào list.

Bucket Array:
┌─────┬──────────────────────────────────┐
│  0  │ null                             │
│  1  │ (15,"A")→(22,"B")→null          │  ← chain of 2
│  2  │ (9,"C")→null                     │
│  3  │ null                             │
│  4  │ (25,"D")→(11,"E")→(32,"F")→null │  ← chain of 3
└─────┴──────────────────────────────────┘

Ưu điểm: đơn giản, table không bao giờ "đầy"
Nhược điểm: extra memory (node pointers), cache-unfriendly (linked list rải rác)
```

### 3.2 Open Addressing (Không dùng linked list)

Khi collision, tìm slot **khác trong cùng array**:

**Linear Probing**: Tìm slot kế tiếp `(hash + 1) % size, (hash + 2) % size, ...`

```
Insert: hash("Alice")=2, hash("Bob")=2 (collision!)

  Step 1: slot 2 đã có Alice → thử slot 3
  Step 2: slot 3 trống → đặt Bob ở slot 3

┌────┬────────┬────────┬────────┬────────┐
│  0 │ empty  │ empty  │ Alice  │  Bob   │
└────┴────────┴────────┴────────┴────────┘
                 [2]       [3]
                 hash=2    probed to 3

Vấn đề: PRIMARY CLUSTERING — các phần tử tụ lại thành cụm
         → tìm kiếm chậm dần
```

**Quadratic Probing**: `(hash + 1²), (hash + 2²), (hash + 3²), ...`

- Giảm clustering nhưng có thể không duyệt hết bảng

**Double Hashing**: `(hash1 + i × hash2) % size`

- Tốt nhất cho open addressing — phân bố đều nhất

### 3.3 So sánh

|                     | Separate Chaining | Open Addressing      |
| ------------------- | ----------------- | -------------------- |
| **Load factor > 1** | ✅ Được           | ❌ Không thể         |
| **Clustering**      | Không             | Có (linear)          |
| **Cache**           | Kém (linked list) | Tốt (contiguous)     |
| **Delete**          | Đơn giản          | Phức tạp (tombstone) |
| **Implementation**  | Dễ                | Khó hơn              |
| **Dùng bởi**        | Java HashMap      | Python dict, V8 Map  |

## 4. Load Factor & Rehashing

```
Load Factor (α) = n / m
  n = số phần tử đang lưu
  m = số buckets

  α cao → nhiều collision → chậm
  α thấp → ít collision nhưng lãng phí memory

Ngưỡng rehash thường dùng:
  Chaining: α > 0.75 (Java HashMap default)
  Open Addressing: α > 0.5 - 0.7

Rehashing process:
  1. Tạo bảng mới, capacity × 2 (thường)
  2. Re-hash TẤT CẢ entries (vì hash = key % NEW_SIZE thay đổi!)
  3. Insert lại vào bảng mới
  4. Free bảng cũ

  Cost: O(n) mỗi lần rehash
  Amortized: O(1) per insert (giống dynamic array doubling)
```

## 5. Implementation Từ Đầu

```javascript
class HashTable {
  constructor(initialCapacity = 16) {
    this.buckets = new Array(initialCapacity).fill(null).map(() => []);
    this.size = 0;
    this.capacity = initialCapacity;
    this.LOAD_THRESHOLD = 0.75;
  }

  _hash(key) {
    let hash = 0;
    const str = String(key);
    for (let i = 0; i < str.length; i++) {
      hash = (hash * 31 + str.charCodeAt(i)) % this.capacity;
    }
    return hash;
  }

  set(key, value) {
    if (this.size / this.capacity >= this.LOAD_THRESHOLD) {
      this._rehash();
    }
    const index = this._hash(key);
    const bucket = this.buckets[index];
    // Check if key already exists → update
    for (const entry of bucket) {
      if (entry[0] === key) {
        entry[1] = value;
        return;
      }
    }
    // New key → insert
    bucket.push([key, value]);
    this.size++;
  }

  get(key) {
    const index = this._hash(key);
    for (const entry of this.buckets[index]) {
      if (entry[0] === key) return entry[1];
    }
    return undefined;
  }

  delete(key) {
    const index = this._hash(key);
    const bucket = this.buckets[index];
    for (let i = 0; i < bucket.length; i++) {
      if (bucket[i][0] === key) {
        bucket.splice(i, 1);
        this.size--;
        return true;
      }
    }
    return false;
  }

  _rehash() {
    const oldBuckets = this.buckets;
    this.capacity *= 2;
    this.buckets = new Array(this.capacity).fill(null).map(() => []);
    this.size = 0;
    for (const bucket of oldBuckets) {
      for (const [key, value] of bucket) {
        this.set(key, value); // re-hash with new capacity
      }
    }
  }
}
```

## 6. JavaScript Map vs Object

```javascript
// Object — basic hash map, nhưng có quirks
const obj = {};
obj["key"] = "value";
// Vấn đề:
//   - Keys chỉ là String hoặc Symbol
//   - Có prototype pollution risk
//   - Không biết size trực tiếp (Object.keys(obj).length)
//   - Iteration order: integer keys sorted, then string keys insertion order

// Map — proper hash map (ES6+)
const map = new Map();
map.set("key", "value");
map.set(42, "number key"); // key là BẤT KỲ type
map.set({ a: 1 }, "object key"); // object as key (by reference!)
map.size; // O(1) size
// Iteration order: LUÔN insertion order
// Performance: tốt hơn Object cho frequent add/delete

// ✅ Dùng Map khi:
//   - Keys không phải string
//   - Cần .size
//   - Frequent add/delete
//   - Cần deterministic iteration order

// ✅ Dùng Object khi:
//   - JSON serialization (Map doesn't serialize)
//   - Simple key-value với string keys
//   - Accessing object properties
```

| Feature                  | Object                        | Map                                      |
| ------------------------ | ----------------------------- | ---------------------------------------- |
| Key types                | String, Symbol                | **Bất kỳ** (object, function, primitive) |
| Size                     | `Object.keys().length` (O(n)) | `.size` (O(1))                           |
| Iteration                | Không guaranteed order        | **Insertion order**                      |
| Performance (add/delete) | Chậm hơn                      | **Nhanh hơn**                            |
| JSON                     | ✅ Native                     | ❌ Cần custom serializer                 |
| Default keys             | Có (prototype)                | Không                                    |

## 7. Hash Set

Hash Set = Hash Map nhưng **chỉ lưu key** (không có value). Dùng để kiểm tra membership trong O(1).

```javascript
const set = new Set();
set.add(1);
set.add(2);
set.add(3);
set.has(2); // true  — O(1)
set.delete(2); // true
set.size; // 2

// Use cases phổ biến:
// 1. Loại bỏ duplicates
const unique = [...new Set([1, 2, 2, 3, 3, 3])]; // [1, 2, 3]

// 2. Check intersection
const setA = new Set([1, 2, 3]);
const setB = new Set([2, 3, 4]);
const intersect = [...setA].filter((x) => setB.has(x)); // [2, 3]
```

## 8. Patterns Phỏng Vấn

**Two Sum — Bài kinh điển nhất (LC #1)**:

```javascript
function twoSum(nums, target) {
  const map = new Map(); // value → index
  for (let i = 0; i < nums.length; i++) {
    const complement = target - nums[i];
    if (map.has(complement)) return [map.get(complement), i];
    map.set(nums[i], i);
  }
  return [-1, -1];
}
// O(n) time, O(n) space — dùng hash map để lookup complement trong O(1)
```

**Group Anagrams (LC #49)**:

```javascript
function groupAnagrams(strs) {
  const map = new Map();
  for (const str of strs) {
    const key = str.split("").sort().join(""); // sorted = anagram key
    if (!map.has(key)) map.set(key, []);
    map.get(key).push(str);
  }
  return [...map.values()];
}
// "eat","tea","ate" → sorted key = "aet" → cùng group
```

**Frequency Counter Pattern**:

```javascript
// Đếm tần suất — dùng cho rất nhiều bài
function topKFrequent(nums, k) {
  const freq = new Map();
  for (const n of nums) freq.set(n, (freq.get(n) || 0) + 1);

  // Bucket sort by frequency — O(n)
  const buckets = new Array(nums.length + 1).fill(null).map(() => []);
  for (const [num, count] of freq) buckets[count].push(num);

  const result = [];
  for (let i = buckets.length - 1; i >= 0 && result.length < k; i--) {
    result.push(...buckets[i]);
  }
  return result;
}
```

**Longest Substring Without Repeating Characters (LC #3)**:

```javascript
function lengthOfLongestSubstring(s) {
  const map = new Map(); // char → last seen index
  let maxLen = 0,
    left = 0;
  for (let right = 0; right < s.length; right++) {
    if (map.has(s[right]) && map.get(s[right]) >= left) {
      left = map.get(s[right]) + 1; // shrink window
    }
    map.set(s[right], right);
    maxLen = Math.max(maxLen, right - left + 1);
  }
  return maxLen;
}
// Sliding Window + Hash Map — O(n)
```

## 9. Câu Hỏi Phỏng Vấn

**Q: Hash Table O(1) lookup — thật sự O(1) không?**

> A: **Average O(1)**, worst case O(n) khi tất cả keys hash vào cùng bucket (trở thành linked list). Giữ load factor thấp (< 0.75) và dùng good hash function để worst case hiếm xảy ra. Java 8+ chuyển chaining sang Red-Black Tree khi chain > 8 → worst case O(log n).

**Q: Chaining vs Open Addressing — khi nào dùng cái nào?**

> A: **Chaining**: đơn giản, load factor > 1 OK, delete dễ. Dùng khi không biết trước số elements. **Open Addressing**: cache-friendly hơn (contiguous array), nhưng load factor phải < 1, delete cần tombstone. Python dict và V8 Map dùng open addressing vì performance tốt hơn cho workloads thực tế.

**Q: Tại sao table size nên là số nguyên tố?**

> A: Khi dùng `hash % size`, nếu size = 2^k thì chỉ dùng k bits thấp của hash → nếu hash function có pattern ở bits thấp → clustering. Số nguyên tố phân bố đều hơn vì không có common factor với phần lớn hash values. Tuy nhiên, nhiều modern implementations dùng power-of-2 size với Fibonacci hashing để tối ưu modulo bằng bitwise AND.
