## 1. gRPC Là Gì

gRPC (Google Remote Procedure Call) — framework để gọi function trên remote server **như thể gọi local function**.

```
Truyền thống (REST):
  Client: POST /api/users { "name": "Alice" }
  → Construct URL, serialize JSON, parse response JSON
  → "HTTP-centric" thinking

gRPC:
  Client: userService.CreateUser({ name: "Alice" })
  → Gọi function trực tiếp! Framework lo serialization, transport
  → "Method call" thinking — giống gọi function local

Bên trong:
  - Data: Protocol Buffers (binary, nhỏ hơn JSON 3-10x)
  - Transport: HTTP/2 (multiplexing, streaming)
  - Contract: .proto file (strongly typed interface)
```

## 2. Protocol Buffers — Protobuf

### 2.1 Định nghĩa Schema (.proto file)

```protobuf
syntax = "proto3";

package user;

// Service definition — API contract
service UserService {
  rpc GetUser (GetUserRequest) returns (User);
  rpc CreateUser (CreateUserRequest) returns (User);
  rpc ListUsers (ListUsersRequest) returns (stream User); // server streaming
  rpc Chat (stream ChatMessage) returns (stream ChatMessage); // bidirectional
}

// Message definitions — data structure
message User {
  int32 id = 1;          // field number (không thay đổi sau khi deploy!)
  string name = 2;
  string email = 3;
  Role role = 4;
  repeated string tags = 5; // array
}

enum Role {
  UNKNOWN = 0;
  ADMIN = 1;
  USER = 2;
}

message GetUserRequest {
  int32 id = 1;
}

message CreateUserRequest {
  string name = 1;
  string email = 2;
}

message ListUsersRequest {
  int32 page = 1;
  int32 page_size = 2;
}

message ChatMessage {
  string user = 1;
  string text = 2;
  int64 timestamp = 3;
}
```

### 2.2 Protobuf vs JSON

```
JSON (text-based):
  {"id": 1, "name": "Alice", "email": "alice@test.com", "role": "ADMIN"}
  Size: ~70 bytes. Parse time: ~1ms

Protobuf (binary):
  08 01 12 05 41 6C 69 63 65 1A 0E 61 6C 69 63 65...
  Size: ~25 bytes (3x nhỏ hơn!). Parse time: ~0.1ms (10x nhanh hơn!)

Tại sao nhỏ hơn?
  - Field names thay bằng field numbers (1, 2, 3...)
  - Binary encoding: int32 dùng varint, không phải text
  - Không có quotes, colons, brackets
```

| Feature                | JSON          | Protobuf              |
| ---------------------- | ------------- | --------------------- |
| Format                 | Text          | Binary                |
| Size                   | Lớn           | **3-10x nhỏ hơn**     |
| Parse speed            | Chậm          | **5-100x nhanh**      |
| Human-readable         | ✅            | ❌                    |
| Schema                 | Optional      | **Bắt buộc (.proto)** |
| Backward compatibility | Không đảm bảo | ✅ Field numbers      |
| Language support       | Universal     | Code generation       |

## 3. 4 Streaming Types

```
1. UNARY — Đơn giản nhất (như REST)
   Client ── Request ──▶ Server
   Client ◀── Response ── Server

2. SERVER STREAMING — Server gửi nhiều responses
   Client ── Request ──▶ Server
   Client ◀── Response 1 ── Server
   Client ◀── Response 2 ── Server
   Client ◀── Response 3 ── Server
   Use case: Download large dataset, real-time updates

3. CLIENT STREAMING — Client gửi nhiều requests
   Client ── Request 1 ──▶ Server
   Client ── Request 2 ──▶
   Client ── Request 3 ──▶
   Client ◀── Response ── Server
   Use case: File upload, batch processing

4. BIDIRECTIONAL STREAMING — Cả hai gửi song song
   Client ── Request 1 ──▶ Server
   Client ◀── Response 1 ── Server
   Client ── Request 2 ──▶
   Client ── Request 3 ──▶
   Client ◀── Response 2 ── Server
   Use case: Chat, collaborative editing, gaming
```

## 4. gRPC vs REST

| Feature         | REST                       | gRPC                  |
| --------------- | -------------------------- | --------------------- |
| Protocol        | HTTP/1.1 (thường)          | HTTP/2                |
| Data format     | JSON (text)                | Protobuf (binary)     |
| Contract        | OpenAPI/Swagger (optional) | **.proto (bắt buộc)** |
| Performance     | Chậm hơn                   | **3-10x nhanh**       |
| Streaming       | Không native               | **4 types**           |
| Browser support | ✅ Native                  | ❌ Cần grpc-web proxy |
| Code generation | Optional                   | ✅ Built-in           |
| Human-readable  | ✅                         | ❌                    |
| Learning curve  | Thấp                       | Trung bình            |

### Khi nào dùng?

```
REST:
  ✅ Public APIs (browser clients)
  ✅ Simple CRUD
  ✅ Third-party integration (everyone knows REST)
  ✅ Cần human-readable debugging

gRPC:
  ✅ Microservice-to-microservice communication
  ✅ High-performance, low latency
  ✅ Streaming (real-time data)
  ✅ Polyglot systems (nhiều languages, code gen từ .proto)
  ✅ Internal APIs at Big Tech (Google, Netflix, Uber)
```

## 5. Node.js Implementation

```javascript
// server.js
const grpc = require("@grpc/grpc-js");
const protoLoader = require("@grpc/proto-loader");

const packageDef = protoLoader.loadSync("./user.proto");
const proto = grpc.loadPackageDefinition(packageDef);

const users = [
  { id: 1, name: "Alice", email: "alice@test.com" },
  { id: 2, name: "Bob", email: "bob@test.com" },
];

const server = new grpc.Server();
server.addService(proto.user.UserService.service, {
  GetUser: (call, callback) => {
    const user = users.find((u) => u.id === call.request.id);
    if (user) callback(null, user);
    else callback({ code: grpc.status.NOT_FOUND, message: "User not found" });
  },
  ListUsers: (call) => {
    // Server streaming
    users.forEach((user) => call.write(user));
    call.end();
  },
});
server.bindAsync(
  "0.0.0.0:50051",
  grpc.ServerCredentials.createInsecure(),
  () => {
    console.log("gRPC server running on port 50051");
  },
);

// client.js
const client = new proto.user.UserService(
  "localhost:50051",
  grpc.credentials.createInsecure(),
);

// Unary call — giống gọi function!
client.GetUser({ id: 1 }, (err, user) => {
  console.log(user); // { id: 1, name: 'Alice', email: 'alice@test.com' }
});

// Server streaming
const stream = client.ListUsers({ page: 1, page_size: 10 });
stream.on("data", (user) => console.log("User:", user));
stream.on("end", () => console.log("Done"));
```

## 6. Câu Hỏi Phỏng Vấn

**Q: gRPC vs REST — khi nào dùng gRPC?**

> A: gRPC cho **internal microservice** communication: binary protobuf (3-10x nhỏ hơn JSON), HTTP/2 multiplexing, bidirectional streaming, code generation từ .proto schema. REST cho **public-facing APIs**: browser-friendly, human-readable, ecosystem lớn. Big Tech (Google, Netflix) dùng gRPC internal, REST external.

**Q: Protocol Buffers advantages over JSON?**

> A: (1) **Smaller**: binary encoding, field numbers thay field names → 3-10x nhỏ hơn. (2) **Faster**: no text parsing → 5-100x nhanh serialize/deserialize. (3) **Schema enforcement**: .proto file = contract → catch errors compile-time. (4) **Backward compatible**: thêm fields mới mà không break clients (field numbers giữ nguyên). Nhược: không human-readable, cần tooling.
