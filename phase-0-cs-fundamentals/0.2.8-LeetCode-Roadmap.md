## 1. Tổng Quan Strategy

```
❌ SAI: Giải random 500 bài → nhớ solution → quên sau 2 tuần
✅ ĐÚNG: Học 14 patterns → nhận diện bài → áp template → biến thể

Pattern-based approach:
  1. Học pattern (ví dụ: Sliding Window)
  2. Giải 5-8 bài cùng pattern
  3. Tổng kết: khi nào dùng, template, edge cases
  4. Spaced repetition: ôn lại sau 1 ngày, 3 ngày, 1 tuần
```

## 2. 14 Patterns Cốt Lõi

### Pattern 1: Two Pointers

```
Khi nào: Sorted array, tìm pairs, remove duplicates
Template: left=0, right=n-1, di chuyển dựa trên condition
Bài:     LC #1, #15, #11, #167, #125, #42
```

### Pattern 2: Sliding Window

```
Khi nào: Substring/subarray tối ưu, consecutive elements
Template: left (start), right (expand), shrink khi violate condition
Bài:     LC #3, #76, #209, #424, #567
```

### Pattern 3: Fast & Slow Pointers

```
Khi nào: Linked list cycle, middle, palindrome
Template: slow=1step, fast=2steps
Bài:     LC #141, #142, #876, #234, #287
```

### Pattern 4: Merge Intervals

```
Khi nào: Overlapping intervals, schedule
Template: Sort by start, merge if overlapping
Bài:     LC #56, #57, #252, #253, #435
```

### Pattern 5: Binary Search

```
Khi nào: Sorted data, search space có thể chia đôi
Template: left, right, mid, shrink based on condition
Bài:     LC #704, #33, #34, #153, #875, #1011
```

### Pattern 6: BFS / Level-order

```
Khi nào: Shortest path (unweighted), level-by-level
Template: queue, visited set, process level by level
Bài:     LC #102, #200, #994, #127, #752
```

### Pattern 7: DFS / Backtracking

```
Khi nào: All possibilities, paths, permutations
Template: choose-explore-unchoose, visited tracking
Bài:     LC #78, #46, #39, #79, #51, #17
```

### Pattern 8: Stack (Monotonic)

```
Khi nào: Next greater/smaller, rectangle in histogram
Template: stack maintains increasing/decreasing order
Bài:     LC #739, #84, #42, #496, #503, #907
```

### Pattern 9: Hash Map / Set

```
Khi nào: Lookup O(1), frequency, duplicates
Template: build map first, then query
Bài:     LC #1, #49, #128, #347, #560
```

### Pattern 10: Dynamic Programming

```
Khi nào: Optimization, count ways, overlapping subproblems
Template: define state, recurrence, base case
Bài:     LC #70, #198, #300, #322, #1143, #62
```

### Pattern 11: Greedy

```
Khi nào: Local optimal = global optimal
Template: sort + iterate, make best choice each step
Bài:     LC #55, #45, #435, #763, #621
```

### Pattern 12: Heap / Priority Queue

```
Khi nào: Top K, merge sorted, scheduling
Template: min/max heap, maintain size K
Bài:     LC #215, #295, #347, #23, #973
```

### Pattern 13: Trie

```
Khi nào: Prefix matching, autocomplete, word search
Template: children map, isEnd flag
Bài:     LC #208, #211, #212, #648
```

### Pattern 14: Union-Find

```
Khi nào: Connected components, cycle detection (undirected)
Template: parent[], find with path compression, union by rank
Bài:     LC #200, #547, #323, #684, #721
```

```

━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Mục tiêu: nắm vững data structures, viết code sạch, xử lý edge cases
  Thời gian/bài: 15-30 phút

Tháng 3-4: MEDIUM (150-200 bài)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  Tuần 1-2: Tree DFS/BFS + BST (25 bài)

  Tuần 5-6: Dynamic Programming 1D + 2D (30 bài)
  Tuần 7-8: Backtracking + Greedy (20 bài)

  Mục tiêu: nhận diện patterns, tối ưu từ brute force
  Thời gian/bài: 30-45 phút

Tháng 5-6: HARD + CONTEST (50-80 bài)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Song song: Ôn bài cũ (spaced repetition)
  Ngày 1 → Ngày 3 → Ngày 7 → Ngày 14 → Ngày 30
```

## 4. Cách Giải 1 Bài LeetCode Hiệu Quả

```
Bước 1: ĐỌC KỸ (2 phút)
  - Đọc đề 2 lần
  - Highlight constraints (n ≤ 10^5 → cần O(n log n) trở xuống)
  - Viết ra examples thêm, đặc biệt edge cases

Bước 2: SUY NGHĨ APPROACH (5 phút)
  - Brute force trước → analyze complexity
  - Nhận diện pattern dựa trên dấu hiệu:
    "Sorted array" → Two Pointers / Binary Search
    "Substring/subarray" → Sliding Window
    "Count ways / min-max" → DP
    "All combinations" → Backtracking
    "Connected components" → Union-Find / DFS
    "Top K" → Heap
  - Nói ra approach trước khi code (phỏng vấn!)

Bước 3: CODE (15-25 phút)
  - Code clean, đặt tên biến rõ ràng
  - Comment ý tưởng chính

Bước 4: TEST (3-5 phút)
  - Dry-run với example
  - Test edge cases: empty, single element, all same, sorted, reverse sorted

Bước 5: OPTIMIZE (5 phút)
  - Có thể giảm time/space?
  - Discuss trade-offs

Tổng: ≤45 phút
Nếu stuck > 20 phút → xem hint / editorial → HỌC pattern → giải lại ngày hôm sau
```
