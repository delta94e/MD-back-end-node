## 1. DNS Là Gì

DNS = "Danh bạ của Internet" — chuyển đổi tên miền → IP address.

```
Bạn gõ: google.com
DNS trả: 142.250.185.78
Browser kết nối đến: 142.250.185.78:443 (HTTPS)

Tại sao cần? → Con người nhớ "google.com", máy tính cần IP address.
```

## 2. DNS Resolution Flow

```
  Browser              Local DNS              Root DNS    TLD DNS     Authoritative
  (Client)             Resolver               Server      (.com)      (google.com)
    │                     │                      │           │              │
    │ "google.com?" ────▶ │                      │           │              │
    │                     │                      │           │              │
    │                     │ Check cache           │           │              │
    │                     │ (miss!)               │           │              │
    │                     │                      │           │              │
    │                     │ "google.com?" ──────▶ │           │              │
    │                     │                      │           │              │
    │                     │ ◀── ".com DNS is     │           │              │
    │                     │      192.5.6.30"     │           │              │
    │                     │                      │           │              │
    │                     │ "google.com?" ──────────────────▶ │              │
    │                     │                      │           │              │
    │                     │ ◀── "google.com DNS is ──────── │              │
    │                     │      216.239.32.10"  │           │              │
    │                     │                      │           │              │
    │                     │ "google.com?" ──────────────────────────────▶  │
    │                     │                      │           │              │
    │                     │ ◀── "142.250.185.78" ──────────────────────── │
    │                     │                      │           │              │
    │                     │ CACHE result (TTL)    │           │              │
    │                     │                      │           │              │
    │ ◀── 142.250.185.78  │                      │           │              │

Recursive: Resolver hỏi lần lượt (Root → TLD → Authoritative)
Iterative: Mỗi server trả "hỏi server kia đi" (referral)
```

## 3. DNS Record Types

| Record    | Mô tả                     | Ví dụ                                                |
| --------- | ------------------------- | ---------------------------------------------------- |
| **A**     | Domain → IPv4             | `google.com → 142.250.185.78`                        |
| **AAAA**  | Domain → IPv6             | `google.com → 2607:f8b0:4004::`                      |
| **CNAME** | Alias → canonical name    | `www.google.com → google.com`                        |
| **MX**    | Mail server               | `google.com → smtp.google.com (priority 10)`         |
| **NS**    | Nameserver cho domain     | `google.com → ns1.google.com`                        |
| **TXT**   | Text data                 | SPF, DKIM (email auth), domain verification          |
| **SRV**   | Service discovery         | `_sip._tcp.example.com → sipserver.example.com:5060` |
| **SOA**   | Start of Authority        | TTL, refresh, retry, expire                          |
| **PTR**   | IP → Domain (reverse DNS) | `78.185.250.142 → google.com`                        |

## 4. DNS Caching & TTL

```
Caching hierarchy:
  1. Browser cache (vài phút)
  2. OS cache (/etc/hosts, Windows DNS cache)
  3. Local DNS Resolver cache (ISP)
  4. Authoritative server (source of truth)

TTL (Time To Live):
  DNS record có TTL = bao lâu cache valid
  TTL = 300 → cache 5 phút
  TTL = 3600 → cache 1 giờ
  TTL = 86400 → cache 1 ngày

Thấp TTL:
  ✅ Thay đổi IP nhanh (failover, migration)
  ❌ Nhiều DNS queries hơn → chậm hơn

Cao TTL:
  ✅ Ít queries, nhanh hơn
  ❌ Thay đổi IP mất lâu propagate
```

```bash
# Xem DNS records
dig google.com A           # IPv4
dig google.com AAAA        # IPv6
dig google.com MX          # Mail servers
dig google.com NS          # Nameservers
nslookup google.com        # Simple lookup

# Xem TTL
dig google.com A +short    # Chỉ IP
dig google.com A           # Full output bao gồm TTL
```

## 5. DNS cho Backend

| Use Case              | Cách dùng DNS                                             |
| --------------------- | --------------------------------------------------------- |
| **Load Balancing**    | Round-robin DNS: trả nhiều IPs, client chọn ngẫu nhiên    |
| **CDN Routing**       | GeoDNS: trả IP CDN edge gần user nhất                     |
| **Failover**          | Health check → remove unhealthy server IPs, thấp TTL      |
| **Service Discovery** | SRV records cho microservices, hoặc internal DNS (Consul) |
| **Blue-Green Deploy** | Đổi DNS record từ Blue → Green server                     |

```javascript
// Node.js DNS
const dns = require("dns");

// dns.lookup() — dùng OS resolver (có cache, blocking — thread pool!)
dns.lookup("google.com", (err, address) => {
  console.log(address); // 142.250.185.78
});

// dns.resolve() — dùng network DNS trực tiếp (no OS cache, non-blocking)
dns.resolve4("google.com", (err, addresses) => {
  console.log(addresses); // ['142.250.185.78', '142.250.185.100', ...]
});

// ⚠️ dns.lookup() dùng thread pool! Nếu DNS chậm → block thread pool
// → Ảnh hưởng fs operations! Fix: tăng UV_THREADPOOL_SIZE
```

## 6. Câu Hỏi Phỏng Vấn

**Q: DNS resolution diễn ra thế nào?**

> A: Browser check cache → OS cache → Local DNS Resolver. Nếu miss: Resolver hỏi Root DNS (trả TLD server) → TLD DNS (trả Authoritative server) → Authoritative DNS (trả IP). Kết quả được cache ở mỗi layer với TTL. Dùng UDP port 53 (nhanh, query nhỏ). Nếu response > 512 bytes → fallback TCP.

**Q: CNAME vs A record — khi nào dùng?**

> A: **A record**: trỏ trực tiếp đến IP. Dùng cho root domain (example.com). **CNAME**: alias trỏ đến domain khác. Dùng cho subdomains (www.example.com → example.com, hoặc cdn.example.com → d123.cloudfront.net). CNAME cho root domain không được chuẩn RFC. Một số providers dùng ALIAS/ANAME record thay thế.
