## 1. Định Nghĩa Cơ Bản

```
Graph G = (V, E)
  V = tập đỉnh (vertices / nodes)
  E = tập cạnh (edges)

Phân loại:
                Undirected          Directed (DiGraph)
                A ─── B             A ───▶ B
                │     │             │      │
                C ─── D             C ◀─── D

                Weighted             Unweighted
                A ──5── B           A ─── B
                │       │           │     │
                3       4           C ─── D
                │       │
                C ──1── D

V = {A, B, C, D}
E_undirected = {(A,B), (A,C), (B,D), (C,D)}
E_directed   = {(A,B), (A,C), (D,C)}      ← có hướng!
```

## 2. Degree — Bậc Của Đỉnh

```
Undirected Graph:
  degree(v) = số cạnh nối với v

       A ─── B ─── E
       │   / │
       C ─── D

  degree(A) = 2 (nối B, C)
  degree(B) = 4 (nối A, C, D, E)
  degree(C) = 3 (nối A, B, D)
  degree(D) = 2 (nối B, C)
  degree(E) = 1 (nối B)

  Handshaking Theorem: Σ degree(v) = 2 × |E|
  → Tổng bậc = 2 × số cạnh (mỗi cạnh đóng góp 2 bậc)

Directed Graph:
  in-degree(v)  = số cạnh ĐẾN v
  out-degree(v) = số cạnh ĐI TỪ v

       A ──▶ B ──▶ E
       │   ▲ │
       ▼  /  ▼
       C ──▶ D

  in-degree(B) = 2 (A→B, C→B)
  out-degree(B) = 2 (B→D, B→E)

Ứng dụng:
  Topological Sort: bắt đầu từ nodes có in-degree = 0
  Social Network: degree = số bạn bè (follower count = in-degree)
```

## 3. Path & Cycle

```
PATH: chuỗi vertices liên tiếp nối bởi edges
  A → B → D → C = path dài 3 (3 edges)

SIMPLE PATH: không đi qua vertex nào 2 lần
  A → B → D ✓ (simple)
  A → B → C → A → D ✗ (A visited twice, not simple)

CYCLE: path bắt đầu VÀ kết thúc tại cùng vertex
  A → B → C → A = cycle length 3

  Undirected: A ─ B ─ C ─ A (min cycle length = 3)
  Directed:   A → B → C → A
  Self-loop:  A → A (cycle length 1)

  ┌───────────┐
  ▼           │
  A ──▶ B ──▶ C     ← Cycle: A → B → C → A

ACYCLIC: KHÔNG có cycle
  Tree = connected acyclic undirected graph
  DAG  = directed acyclic graph

Cycle Detection:
  Undirected: DFS, nếu gặp visited node (không phải parent) → cycle!
  Directed: DFS, nếu gặp node đang trong current path (GRAY) → cycle!
  → Dùng Union-Find (undirected) hoặc DFS colors (directed)
```

## 4. Connectivity — Thành Phần Liên Thông

```
Connected (undirected):
  Mọi vertex reachable từ mọi vertex khác

  Connected:               NOT Connected:
  A ─── B                  A ─── B    D ─── E
  │     │                  │          │
  C ─── D                  C          F

  → 1 connected component   → 2 connected components:
                              {A, B, C} và {D, E, F}

Strongly Connected (directed):
  Mọi vertex reachable từ mọi vertex khác THEO HƯỚNG

  Strongly Connected:       NOT Strongly Connected:
  A ──▶ B                   A ──▶ B ──▶ C
  ▲     │                   (A→C OK, nhưng C→A? Không!)
  │     ▼
  C ◀── D

Ứng dụng:
  Network: "server nào không reachable?" → connected components
  Social: "friendship groups" → connected components
  Deadlock: "circular dependency?" → cycle in directed graph
```

## 5. DAG — Directed Acyclic Graph

```
DAG = Directed graph KHÔNG có cycle

  ✅ DAG:                    ❌ Không phải DAG (có cycle):
  A ──▶ B ──▶ D              A ──▶ B
  │     │                    ▲     │
  ▼     ▼                    │     ▼
  C ──▶ E                    D ◀── C

DAG Properties:
  1. Luôn có ÍT NHẤT 1 vertex với in-degree = 0 (source)
  2. Luôn có ÍT NHẤT 1 vertex với out-degree = 0 (sink)
  3. Luôn có TOPOLOGICAL ORDER (xếp thứ tự phụ thuộc)
```

## 6. Đặc Biệt: Trees vs Graphs

```
TREE = Connected Acyclic Undirected Graph

Tree properties:
  1. |E| = |V| - 1 (edges = vertices - 1)
  2. Giữa 2 vertices BẤT KỲ chỉ có ĐÚNG 1 path
  3. Thêm 1 edge → tạo cycle
  4. Xóa 1 edge → disconnected

Tree ⊂ Graph (tree là trường hợp đặc biệt của graph)

  Tree:                  Graph (with cycle):
       A                      A ─── B
      / \                     │   / │
     B   C                    │  /  │
    / \                       C ─── D
   D   E
  |E| = 4 = |V| - 1 ✓        |E| = 5 > |V| - 1 → có cycle!

Spanning Tree:
  Subgraph là tree chứa TẤT CẢ vertices
  Minimum Spanning Tree (MST): spanning tree với tổng weight nhỏ nhất
  → Kruskal's, Prim's algorithm
```

## 8. Câu Hỏi Phỏng Vấn

**Q: Tree vs Graph — khác gì? Khi nào dùng?**

> A: **Tree** = connected acyclic graph, |E| = |V| - 1, có unique path giữa mọi cặp vertices. Dùng: hierarchical data (DOM, file system, org chart). **Graph** = tổng quát hơn, có thể có cycles, multiple paths, disconnected components. Dùng: social networks, maps, dependencies. Mọi tree đều là graph, nhưng không phải mọi graph là tree.

**Q: DAG vs general graph — tại sao DAG quan trọng?**

> A: DAG = directed + no cycle → có thể **topological sort** (xếp thứ tự phụ thuộc). Rất nhiều real-world problems là DAG: build systems, task scheduling, Git history, data pipelines, course prerequisites. Nếu có cycle → không thể topological sort → deadlock / circular dependency.

**Q: Handshaking Theorem ứng dụng gì?**

> A: Tổng degree = 2 × |E|. (1) Nhanh chóng kiểm tra tính hợp lệ của graph: nếu tổng degree lẻ → graph không hợp lệ. (2) Tính số edges từ degree sequence. (3) Eulerian path tồn tại ⟺ đúng 0 hoặc 2 vertices có degree lẻ.
