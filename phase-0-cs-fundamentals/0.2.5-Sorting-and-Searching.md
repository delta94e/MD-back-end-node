## 1. Tổng Quan Sorting

| Algorithm          | Best       | Average    | Worst      | Space    | Stable? | In-place? |
| ------------------ | ---------- | ---------- | ---------- | -------- | ------- | --------- |
| **Bubble Sort**    | O(n)       | O(n²)      | O(n²)      | O(1)     | ✅      | ✅        |
| **Selection Sort** | O(n²)      | O(n²)      | O(n²)      | O(1)     | ❌      | ✅        |
| **Insertion Sort** | O(n)       | O(n²)      | O(n²)      | O(1)     | ✅      | ✅        |
| **Merge Sort**     | O(n log n) | O(n log n) | O(n log n) | O(n)     | ✅      | ❌        |
| **Quick Sort**     | O(n log n) | O(n log n) | O(n²)      | O(log n) | ❌      | ✅        |
| **Heap Sort**      | O(n log n) | O(n log n) | O(n log n) | O(1)     | ❌      | ✅        |
| **Counting Sort**  | O(n+k)     | O(n+k)     | O(n+k)     | O(k)     | ✅      | ❌        |
| **Radix Sort**     | O(d×n)     | O(d×n)     | O(d×n)     | O(n+k)   | ✅      | ❌        |

**Thuật ngữ**:

- **Stable**: Giữ nguyên thứ tự tương đối của elements bằng nhau. Quan trọng khi sort objects theo multiple keys.
- **In-place**: Không cần thêm memory đáng kể ngoài input (O(1) extra space).

## 2. O(n²) Sorts — Cơ Bản

### Insertion Sort — ĐƠN GIẢN NHẤT & THỰC TẾ NHẤT trong nhóm O(n²)

```javascript
function insertionSort(arr) {
  for (let i = 1; i < arr.length; i++) {
    const key = arr[i];
    let j = i - 1;
    // Dịch chuyển elements lớn hơn key sang phải
    while (j >= 0 && arr[j] > key) {
      arr[j + 1] = arr[j];
      j--;
    }
    arr[j + 1] = key;
  }
  return arr;
}
// Best case O(n): nearly sorted array → chỉ cần vài lần dịch
// Thực tế: rất nhanh cho n < 50 → được dùng làm base case cho Merge/Quick Sort!
```

```
Ví dụ Insertion Sort:
  [5, 2, 4, 6, 1, 3]
   ^sorted|unsorted^

  i=1: key=2, insert vào vị trí đúng → [2, 5, 4, 6, 1, 3]
  i=2: key=4                          → [2, 4, 5, 6, 1, 3]
  i=3: key=6 (đã đúng vị trí)        → [2, 4, 5, 6, 1, 3]
  i=4: key=1, dịch tất cả             → [1, 2, 4, 5, 6, 3]
  i=5: key=3                          → [1, 2, 3, 4, 5, 6] ✓
```

## 3. O(n log n) Sorts — Hiệu Quả

### 3.1 Merge Sort — Stable, Guaranteed O(n log n)

**Ý tưởng**: Divide and Conquer — chia đôi, sort từng nửa, merge lại.

```
       [38, 27, 43, 3, 9, 82, 10]
              /              \
     [38, 27, 43]       [3, 9, 82, 10]
       /      \            /        \
   [38]  [27, 43]     [3, 9]    [82, 10]
          /    \        /  \      /    \
        [27]  [43]    [3]  [9] [82]  [10]
              MERGE ↑
        [27, 43]      [3, 9]    [10, 82]
              MERGE ↑
     [27, 38, 43]       [3, 9, 10, 82]
                 MERGE ↑
       [3, 9, 10, 27, 38, 43, 82]
```

```javascript
function mergeSort(arr) {
  if (arr.length <= 1) return arr;
  const mid = Math.floor(arr.length / 2);
  const left = mergeSort(arr.slice(0, mid));
  const right = mergeSort(arr.slice(mid));
  return merge(left, right);
}

function merge(left, right) {
  const result = [];
  let i = 0,
    j = 0;
  while (i < left.length && j < right.length) {
    if (left[i] <= right[j])
      result.push(left[i++]); // ≤ giữ stability
    else result.push(right[j++]);
  }
  return [...result, ...left.slice(i), ...right.slice(j)];
}
// Time: O(n log n) — always
// Space: O(n) — cần merge buffer
// Stable: YES

// Tại sao O(n log n)?
// log n levels (chia đôi) × O(n) merge mỗi level = O(n log n)
```

### 3.2 Quick Sort — In-place, Fastest in Practice

**Ý tưởng**: Chọn pivot, partition array thành [< pivot, pivot, > pivot], sort từng nửa.

```javascript
function quickSort(arr, low = 0, high = arr.length - 1) {
  if (low >= high) return arr;
  const pivotIdx = partition(arr, low, high);
  quickSort(arr, low, pivotIdx - 1);
  quickSort(arr, pivotIdx + 1, high);
  return arr;
}

function partition(arr, low, high) {
  const pivot = arr[high]; // chọn phần tử cuối làm pivot
  let i = low - 1; // boundary of "< pivot" region
  for (let j = low; j < high; j++) {
    if (arr[j] < pivot) {
      i++;
      [arr[i], arr[j]] = [arr[j], arr[i]]; // swap vào vùng nhỏ
    }
  }
  [arr[i + 1], arr[high]] = [arr[high], arr[i + 1]]; // đặt pivot vào vị trí
  return i + 1;
}

// Average: O(n log n) — nhanh nhất trong thực tế (cache-friendly!)
// Worst: O(n²) — khi pivot luôn là min/max (sorted array + pick last)
// Fix: randomized pivot hoặc median-of-three
```

```
Partition example (pivot = 5):
  [8, 3, 7, 1, 5]
                ^pivot

  j=0: 8≥5 → skip
  j=1: 3<5 → swap → [3, 8, 7, 1, 5], i=0
  j=2: 7≥5 → skip
  j=3: 1<5 → swap → [3, 1, 7, 8, 5], i=1

  Đặt pivot: swap arr[2] và arr[4] → [3, 1, 5, 8, 7]
                                            ^pivot đúng vị trí!
  Tất cả bên trái < 5, bên phải > 5 ✓
```

### 3.3 Merge Sort vs Quick Sort

|                   | Merge Sort                  | Quick Sort                      |
| ----------------- | --------------------------- | ------------------------------- |
| Worst case        | **O(n log n)**              | O(n²)                           |
| Average           | O(n log n)                  | O(n log n)                      |
| Space             | O(n)                        | **O(log n)**                    |
| Stable            | **✅**                      | ❌                              |
| In-place          | ❌                          | **✅**                          |
| Cache performance | Kém (copy arrays)           | **Tốt (in-place partitioning)** |
| Practice          | Linked Lists, external sort | **Arrays (default choice)**     |

### 3.4 Heap Sort

```javascript
function heapSort(arr) {
  const n = arr.length;
  // Build max-heap
  for (let i = Math.floor(n / 2) - 1; i >= 0; i--) heapify(arr, n, i);
  // Extract max, đặt cuối, heapify lại
  for (let i = n - 1; i > 0; i--) {
    [arr[0], arr[i]] = [arr[i], arr[0]];
    heapify(arr, i, 0);
  }
  return arr;
}

function heapify(arr, n, i) {
  let largest = i;
  const left = 2 * i + 1,
    right = 2 * i + 2;
  if (left < n && arr[left] > arr[largest]) largest = left;
  if (right < n && arr[right] > arr[largest]) largest = right;
  if (largest !== i) {
    [arr[i], arr[largest]] = [arr[largest], arr[i]];
    heapify(arr, n, largest);
  }
}
// O(n log n) guaranteed. In-place. NOT stable.
// Ít dùng trong practice vì cache-unfriendly (jump around in array).
```

## 4. Non-Comparison Sorts — O(n)

### Counting Sort — Khi biết range giá trị

```javascript
function countingSort(arr) {
  const max = Math.max(...arr);
  const count = new Array(max + 1).fill(0);

  for (const num of arr) count[num]++;

  let idx = 0;
  for (let i = 0; i <= max; i++) {
    while (count[i] > 0) {
      arr[idx++] = i;
      count[i]--;
    }
  }
  return arr;
}
// O(n + k) time, O(k) space — k = range of values
// Chỉ tốt khi k không quá lớn (ví dụ: sort ages 0-150)
```

## 5. Binary Search — Master Template

### 5.1 Classic Binary Search

```javascript
function binarySearch(arr, target) {
  let left = 0,
    right = arr.length - 1;
  while (left <= right) {
    const mid = left + Math.floor((right - left) / 2); // tránh overflow
    if (arr[mid] === target) return mid;
    if (arr[mid] < target) left = mid + 1;
    else right = mid - 1;
  }
  return -1; // not found
}
// O(log n) — mỗi bước loại bỏ 1/2 search space
```

### 5.2 Template cho mọi biến thể

```javascript
// Tìm vị trí INSERT (leftmost position where condition is true)
function lowerBound(arr, target) {
  let left = 0,
    right = arr.length;
  while (left < right) {
    const mid = left + Math.floor((right - left) / 2);
    if (arr[mid] < target) left = mid + 1;
    else right = mid;
  }
  return left; // first element >= target
}

// Tìm rightmost element ≤ target
function upperBound(arr, target) {
  let left = 0,
    right = arr.length;
  while (left < right) {
    const mid = left + Math.floor((right - left) / 2);
    if (arr[mid] <= target) left = mid + 1;
    else right = mid;
  }
  return left - 1; // last element <= target
}
```

### 5.3 Binary Search on Answer — Pattern rất mạnh

```javascript
// Bài: Koko ăn chuối (LC #875)
// Tìm tốc độ ăn nhỏ nhất để ăn hết trong h giờ
function minEatingSpeed(piles, h) {
  let left = 1,
    right = Math.max(...piles);
  while (left < right) {
    const mid = left + Math.floor((right - left) / 2);
    const hoursNeeded = piles.reduce((sum, p) => sum + Math.ceil(p / mid), 0);
    if (hoursNeeded <= h)
      right = mid; // có thể giảm tốc độ
    else left = mid + 1; // cần tăng tốc độ
  }
  return left;
}
// Binary search trên ANSWER SPACE, không phải trên array!
// Pattern: "tìm giá trị nhỏ nhất/lớn nhất thỏa condition"
```

## 6. Câu Hỏi Phỏng Vấn

**Q: Khi nào dùng Merge Sort vs Quick Sort?**

> A: **Quick Sort** cho general-purpose array sorting (nhanh nhất in practice nhờ cache locality). **Merge Sort** khi cần stability, worst-case guarantee, hoặc sorting linked lists (no random access → QS kém). External sorting (data lớn hơn RAM) dùng Merge Sort.

**Q: Tại sao `Array.sort()` của JS dùng TimSort?**

> A: TimSort = hybrid Merge Sort + Insertion Sort. Tận dụng "runs" (subsequences đã sorted) trong real-world data. Best case O(n) cho nearly sorted data. Stable. V8 chuyển từ Quick Sort sang TimSort vì stability guarantee (ES2019 spec yêu cầu stable sort).

**Q: Binary Search có bao nhiêu biến thể?**

> A: (1) Classic — tìm exact value, (2) Lower Bound — first ≥ target, (3) Upper Bound — first > target, (4) Rotated Array, (5) Search on Answer — minimize/maximize over condition, (6) Matrix Search. Template core giống nhau, chỉ thay đổi condition.
