## 1. Types & Coercion

### 1.1 — 8 Types trong JavaScript

```javascript
// Primitives (7):
typeof undefined; // "undefined"
typeof null; // "object" ← BUG lịch sử! (typeof null === "object")
typeof true; // "boolean"
typeof 42; // "number"
typeof 42n; // "bigint"
typeof "hello"; // "string"
typeof Symbol(); // "symbol"

// Non-primitive (1):
typeof {}; // "object"
typeof []; // "object"  ← Array.isArray([]) = true
typeof function () {}; // "function" ← typeof đặc biệt cho functions

// Kiểm tra chính xác:
Object.prototype.toString.call([]); // "[object Array]"
Object.prototype.toString.call(null); // "[object Null]"
Object.prototype.toString.call(/regex/); // "[object RegExp]"
```

### 1.2 — Abstract Equality (==) Step by Step

```javascript
// == gọi Abstract Equality Comparison Algorithm (spec):
// 1. Same type → so sánh bình thường
// 2. null == undefined → TRUE (đặc biệt!)
// 3. Number vs String → convert String to Number
// 4. Boolean → convert to Number (true→1, false→0)
// 5. Object vs Primitive → call ToPrimitive (valueOf/toString)

"" == false     // "" → 0, false → 0, 0 == 0 → TRUE
"0" == false    // false → 0, "0" → 0, 0 == 0 → TRUE
[] == false     // [] → "" → 0, false → 0, 0 == 0 → TRUE
null == 0       // FALSE! (null chỉ == undefined)
NaN == NaN      // FALSE! (NaN !== NaN, dùng Number.isNaN)

// ✅ LUÔN dùng === trong production code
// === không coerce: khác type → FALSE ngay
```

### 1.3 — Number edge cases

```javascript
0.1 + 0.2 === 0.3; // FALSE! (0.30000000000000004)
// IEEE 754 floating point → binary rounding
// Fix: Math.abs(0.1 + 0.2 - 0.3) < Number.EPSILON

Number.isNaN(NaN); // true
isNaN("hello"); // true ← WRONG! coerces to Number first
Number.isNaN("hello"); // false ← CORRECT

Number.MAX_SAFE_INTEGER; // 2^53 - 1 = 9007199254740991
9007199254740992 === 9007199254740993; // TRUE! ← beyond safe range
// Fix: dùng BigInt cho numbers lớn
```

## 2. Scope & Closures

### 2.1 Lexical Scope

```javascript
// JS dùng LEXICAL (static) scope — quyết định lúc DEFINE, không phải lúc CALL

const x = 10;
function foo() {
  console.log(x); // 10 ← nhìn lên scope chain, tìm x = 10
}

function bar() {
  const x = 20;
  foo(); // 10 ← KHÔNG PHẢI 20! Lexical scope = nơi DEFINE foo
}
bar();
```

### 2.2 Closures — Giải thích từ Memory Level

```javascript
function counter() {
  let count = 0; // captured by closure
  return {
    increment: () => ++count,
    getCount: () => count,
  };
}

const c = counter();
c.increment(); // 1
c.increment(); // 2
c.getCount(); // 2

// Memory model:
// ┌─ counter() EC (đã return, nhưng count KHÔNG bị GC) ─┐
// │  count: 2                                            │
// │     ▲                                                │
// │     │ closure reference                              │
// │  ┌──┴──────────┐   ┌──────────────┐                 │
// │  │ increment() │   │ getCount()   │                 │
// │  │ count → 2   │   │ count → 2    │                 │
// │  └─────────────┘   └──────────────┘                 │
// └──────────────────────────────────────────────────────┘
// → count sống trong memory vì increment/getCount CÒN reference đến nó
```

### 2.3 Closure Gotcha — for loop

```javascript
// ❌ Classic bug:
for (var i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 100); // 3, 3, 3
}
// Tại sao? var = function scope → CÙNG i, setTimeout chạy sau loop → i = 3

// ✅ Fix 1: IIFE
for (var i = 0; i < 3; i++) {
  ((j) => setTimeout(() => console.log(j), 100))(i); // 0, 1, 2
}

// ✅ Fix 2: let (block scope — tạo i mới mỗi iteration)
for (let i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 100); // 0, 1, 2
}
```

## 3. this Keyword

### 4 Rules (theo thứ tự ưu tiên)

```javascript
// Rule 1: new binding (HIGH priority)
function Foo(name) {
  this.name = name;
}
const f = new Foo("Alice"); // this = new object

// Rule 2: Explicit binding (call / apply / bind)
function greet() {
  console.log(this.name);
}
greet.call({ name: "Bob" }); // "Bob"
greet.apply({ name: "Bob" }); // "Bob"
const bound = greet.bind({ name: "Bob" });
bound(); // "Bob"

// Rule 3: Implicit binding (object method call)
const obj = { name: "Charlie", greet };
obj.greet(); // "Charlie" — this = obj

// Rule 4: Default binding (standalone call)
greet(); // undefined (strict mode)
// hoặc global name (sloppy mode)

// ⚠️ Arrow function = KHÔNG CÓ this riêng!
const arrow = () => console.log(this);
// this = lexical this (inherit từ surrounding scope)
// call/apply/bind KHÔNG ảnh hưởng arrow function this!
```

### this traps phổ biến

```javascript
class Timer {
  constructor() {
    this.seconds = 0;
  }
  start() {
    // ❌ setInterval(function() { this.seconds++; }, 1000);
    //    this = global (function tạo this riêng)
    // ✅ setInterval(() => { this.seconds++; }, 1000);
    //    Arrow function kế thừa this từ start() → Timer instance
  }
}
```

## 4. Prototypes

### 4.1 Prototype Chain

```javascript
const animal = { eat: true };
const dog = Object.create(animal);
dog.bark = true;

// dog: { bark: true, [[Prototype]]: animal }
// animal: { eat: true, [[Prototype]]: Object.prototype }
// Object.prototype: { toString, hasOwnProperty, ..., [[Prototype]]: null }

dog.bark; // true  (own property)
dog.eat; // true  (prototype chain → animal)
dog.toString(); // "[object Object]" (prototype chain → Object.prototype)
dog.fly; // undefined (không tìm thấy → chain ends at null)
```

```
  dog                    animal              Object.prototype
  ┌──────────┐          ┌──────────┐         ┌──────────────┐
  │bark: true│          │eat: true │         │toString()    │
  │[[Proto]]─┼────────▶│[[Proto]]─┼────────▶│hasOwnProp()  │
  └──────────┘          └──────────┘         │[[Proto]]: null│
                                              └──────────────┘
```

### 4.2 ES6 Class = Syntactic Sugar

```javascript
// ES6 Class:
class Person {
  constructor(name) {
    this.name = name;
  }
  greet() {
    return `Hi, I'm ${this.name}`;
  }
}

// Equivalent (proof that class = sugar):
function PersonOld(name) {
  this.name = name;
}
PersonOld.prototype.greet = function () {
  return `Hi, I'm ${this.name}`;
};

// Cả hai tạo CÙNG prototype chain!
const p = new Person("Alice");
p.__proto__ === Person.prototype; // true
p.__proto__.__proto__ === Object.prototype; // true
```

## 5. new Operator

```javascript
// new Foo() thực hiện 4 bước:
function myNew(Constructor, ...args) {
  // 1. Tạo object mới
  const obj = {};
  // 2. Set [[Prototype]] = Constructor.prototype
  Object.setPrototypeOf(obj, Constructor.prototype);
  // 3. Gọi Constructor với this = obj mới
  const result = Constructor.apply(obj, args);
  // 4. Nếu Constructor return object → dùng nó. Else → return obj
  return result instanceof Object ? result : obj;
}

// Test:
function Person(name) {
  this.name = name;
}
const p = myNew(Person, "Alice");
p.name; // "Alice"
p instanceof Person; // true
p.__proto__ === Person.prototype; // true ✓
```

## 6. Iterators & Generators

### 6.1 Iterator Protocol

```javascript
// Object là iterable nếu có [Symbol.iterator]() → trả về iterator
// Iterator có method next() → { value, done }

const range = {
  from: 1,
  to: 5,
  [Symbol.iterator]() {
    let current = this.from;
    const last = this.to;
    return {
      next() {
        return current <= last
          ? { value: current++, done: false }
          : { done: true };
      },
    };
  },
};

for (const n of range) console.log(n); // 1, 2, 3, 4, 5
[...range]; // [1, 2, 3, 4, 5] — spread works with iterables!
```

### 6.2 Generators — Lazy Evaluation

```javascript
function* fibonacci() {
  let a = 0,
    b = 1;
  while (true) {
    // INFINITE sequence!
    yield a; // pause, return value
    [a, b] = [b, a + b]; // resume here on next()
  }
}

const fib = fibonacci();
fib.next(); // { value: 0, done: false }
fib.next(); // { value: 1, done: false }
fib.next(); // { value: 1, done: false }
fib.next(); // { value: 2, done: false }
// Lazy: chỉ tính khi CẦN, không tạo infinite array!

// Practical: paginated API fetch
async function* fetchPages(url) {
  let page = 1;
  while (true) {
    const res = await fetch(`${url}?page=${page}`);
    const data = await res.json();
    if (data.length === 0) return;
    yield data;
    page++;
  }
}
```

## 7. Proxy & Reflect

```javascript
// Proxy = intercept operations trên object (meta-programming)
const handler = {
  get(target, prop) {
    console.log(`Accessing ${prop}`);
    return prop in target ? target[prop] : `${prop} not found`;
  },
  set(target, prop, value) {
    if (typeof value !== "number") throw TypeError("Must be number");
    target[prop] = value;
    return true;
  },
};

const obj = new Proxy({}, handler);
obj.x = 42; // OK
obj.x; // "Accessing x" → 42
obj.y; // "Accessing y" → "y not found"
obj.z = "string"; // TypeError: Must be number

// Vue 3 Reactivity dùng Proxy:
// Khi access data → track dependency (get trap)
// Khi set data → trigger re-render (set trap)
```

## 8. Câu Hỏi Phỏng Vấn

**Q: Giải thích Closure với ví dụ thực tế?**

> A: Closure = function "nhớ" lexical scope nơi nó được tạo, ngay cả khi scope đó đã return. Use cases: (1) Data privacy (module pattern), (2) State persistence (counter), (3) Function factories (multiplier(2) → function nhân 2), (4) Partial application/currying. V8 optimize: chỉ capture biến THỰC SỰ DÙNG, không capture entire scope.

**Q: `this` trong arrow function vs regular function?**

> A: Regular function: `this` quyết định bởi **cách gọi** (4 rules). Arrow function: `this` = **lexical this** (kế thừa từ surrounding scope, quyết định lúc DEFINE). Arrow function KHÔNG THỂ bị thay đổi `this` bằng call/apply/bind. Đây là lý do arrow function tốt cho callbacks trong class methods.

**Q: Prototype chain kết thúc ở đâu?**

> A: `Object.prototype.[[Prototype]] === null`. Chain: `obj → Constructor.prototype → Object.prototype → null`. `null` = end of chain. `Object.create(null)` tạo object KHÔNG CÓ prototype — dùng làm "pure dictionary" (không có toString, etc.).
