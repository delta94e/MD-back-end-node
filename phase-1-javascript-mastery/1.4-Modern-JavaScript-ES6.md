## 1. Destructuring

```javascript
// ===== Object Destructuring =====
const user = { name: "Alice", age: 30, address: { city: "HCM" } };

const { name, age } = user; // basic
const { name: userName } = user; // rename
const { role = "user" } = user; // default value
const {
  address: { city },
} = user; // nested

// ===== Array Destructuring =====
const [first, second, ...rest] = [1, 2, 3, 4, 5];
// first=1, second=2, rest=[3,4,5]
const [a, , b] = [1, 2, 3]; // skip: a=1, b=3

// ===== Function Parameters =====
function createUser({ name, role = "user", address: { city } = {} }) {
  return { name, role, city };
}
createUser({ name: "Bob", address: { city: "HN" } });
// Destructure ngay trong parameter — API response parsing rất sạch!

// ===== Swap =====
let x = 1,
  y = 2;
[x, y] = [y, x]; // x=2, y=1 (không cần temp)
```

## 2. Spread & Rest

```javascript
// ===== Spread (...) — Unpack =====
// Array
const arr1 = [1, 2, 3];
const arr2 = [...arr1, 4, 5]; // [1, 2, 3, 4, 5]
const copy = [...arr1]; // SHALLOW copy

// Object
const defaults = { theme: "dark", lang: "en" };
const config = { ...defaults, lang: "vi" }; // { theme: "dark", lang: "vi" }
// ⚠️ Shallow copy: nested objects vẫn là reference!

// Function arguments
Math.max(...[3, 1, 4, 1, 5]); // 5

// ===== Rest (...) — Collect =====
function sum(...nums) {
  // nums = array of args
  return nums.reduce((a, b) => a + b, 0);
}
sum(1, 2, 3, 4); // 10

const { id, ...rest } = { id: 1, name: "A", age: 30 };
// id=1, rest={name:"A", age:30} — tách id ra, giữ phần còn lại
```

## 3. Modules — ESM vs CommonJS

```javascript
// ===== CommonJS (Node.js legacy — synchronous) =====
// module.js
module.exports = { foo: 42, bar: () => {} };
module.exports.baz = "hello";

// main.js
const { foo, bar } = require("./module");
// require() = SYNCHRONOUS, cached sau lần đầu

// ===== ESM (Modern — asynchronous, static) =====
// module.mjs (hoặc "type": "module" trong package.json)
export const foo = 42;
export function bar() {}
export default class MyClass {}

// main.mjs
import MyClass, { foo, bar } from "./module.mjs";
import * as mod from "./module.mjs";
```

### So sánh chi tiết

| Feature            | CommonJS                                     | ESM                                  |
| ------------------ | -------------------------------------------- | ------------------------------------ |
| Syntax             | `require()` / `module.exports`               | `import` / `export`                  |
| Loading            | **Synchronous**                              | **Asynchronous**                     |
| Parsing            | **Dynamic** (runtime)                        | **Static** (compile-time)            |
| Tree Shaking       | ❌ (dynamic → bundler không biết dùng gì)    | ✅ (static → loại bỏ unused exports) |
| Circular Deps      | Partial exports (trả whatever loaded so far) | Live bindings (luôn latest value)    |
| `this` ở top-level | `module.exports`                             | `undefined`                          |
| File extension     | `.js` (default trong Node)                   | `.mjs` hoặc `"type": "module"`       |

```javascript
// Live Bindings (ESM) vs Value Copy (CJS)
// ESM: import nhận REFERENCE đến biến export
// counter.mjs
export let count = 0;
export function increment() {
  count++;
}

// main.mjs
import { count, increment } from "./counter.mjs";
console.log(count); // 0
increment();
console.log(count); // 1 ← live binding! thay đổi theo

// CJS: require nhận VALUE COPY
// Thay đổi trong module KHÔNG reflect ở nơi require
```

## 4. Optional Chaining & Nullish Coalescing

```javascript
// ===== Optional Chaining (?.) =====
const user = { address: { street: "123 Main" } };
user?.address?.street; // "123 Main"
user?.profile?.avatar; // undefined (KHÔNG TypeError!)
// Tương đương: user && user.profile && user.profile.avatar

user?.greet?.(); // gọi method nếu exists, undefined if not
users?.[0]?.name; // array access

// ===== Nullish Coalescing (??) =====
const port = config.port ?? 3000;
// CHỈX null/undefined → dùng default
// 0 ?? 3000 → 0  (0 là giá trị hợp lệ!)
// "" ?? "default" → ""  (chuỗi rỗng hợp lệ!)
// false ?? true → false (false hợp lệ!)

// ⚠️ So sánh với ||
// 0 || 3000 → 3000  ← WRONG! 0 bị coi là falsy
// "" || "default" → "default" ← WRONG!
// false || true → true ← WRONG!

// ?? chỉ check NULL/UNDEFINED → chính xác hơn ||
```

## 5. Map, Set, WeakMap, WeakSet

```javascript
// ===== Map — key-value (any type as key!) =====
const map = new Map();
map.set({ id: 1 }, "user1"); // object as key ✓
map.set(42, "number key"); // number as key ✓
map.size; // 2
map.has(42); // true

// Map vs Object:
// Map: any key type, ordered, .size, iterable, no prototype pollution
// Object: string/symbol keys only, unordered (sort of), JSON-friendly

// ===== Set — unique values =====
const set = new Set([1, 2, 2, 3, 3, 3]);
set.size; // 3 — duplicates removed
set.add(4);
set.has(2); // true
set.delete(2); // true

// Deduplicate array:
const unique = [...new Set([1, 2, 2, 3])]; // [1, 2, 3]

// ===== WeakMap — keys phải là objects, GC-friendly =====
const cache = new WeakMap();
let obj = { data: "heavy" };
cache.set(obj, computeExpensive(obj));
obj = null; // obj không còn reference → WeakMap entry bị GC!
// ✅ Dùng cho: caching, private data, DOM metadata

// ===== WeakSet — objects only, GC-friendly =====
const visited = new WeakSet();
visited.add(domNode);
// domNode bị remove khỏi DOM → tự xóa khỏi WeakSet (no memory leak!)
```

## 6. Symbols

```javascript
// Symbol = primitive UNIQUE & IMMUTABLE
const s1 = Symbol("description");
const s2 = Symbol("description");
s1 === s2; // FALSE! Mỗi Symbol là unique

// ===== Use cases =====
// 1. Private-like properties
const _password = Symbol("password");
class User {
  constructor(name, pass) {
    this.name = name;
    this[_password] = pass; // not enumerable, "hidden"
  }
}
Object.keys(new User("A", "123")); // ["name"] — Symbol không hiện!

// 2. Well-known Symbols — customize JS behavior
class Money {
  constructor(amount) {
    this.amount = amount;
  }
  [Symbol.toPrimitive](hint) {
    if (hint === "number") return this.amount;
    if (hint === "string") return `$${this.amount}`;
    return this.amount;
  }
}
const price = new Money(42);
+price; // 42 (hint: "number")
`${price}`; // "$42" (hint: "string")

// Symbol.iterator — make object iterable (see §1.2)
// Symbol.hasInstance — customize instanceof
// Symbol.species — customize constructor for derived objects
```

## 7. Tagged Templates

```javascript
// Tagged template = function nhận template literal parts
function highlight(strings, ...values) {
  return strings.reduce((result, str, i) => {
    return result + str + (values[i] ? `<b>${values[i]}</b>` : "");
  }, "");
}

const name = "Alice";
const role = "admin";
highlight`User ${name} is ${role}`;
// "User <b>Alice</b> is <b>admin</b>"

// ===== SQL Injection Prevention =====
function sql(strings, ...values) {
  // Escape tất cả interpolated values
  const escaped = values.map((v) =>
    typeof v === "string" ? v.replace(/'/g, "''") : v,
  );
  return strings.reduce((q, str, i) => q + str + (escaped[i] ?? ""), "");
}

const userInput = "'; DROP TABLE users; --";
const query = sql`SELECT * FROM users WHERE name = '${userInput}'`;
// "SELECT * FROM users WHERE name = '''; DROP TABLE users; --'"
// → SQL injection PREVENTED vì quotes đã escaped
```

## 8. Câu Hỏi Phỏng Vấn

**Q: ESM vs CommonJS — khác gì? Tại sao ESM tốt hơn?**

> A: CommonJS: synchronous loading, dynamic imports (runtime), value copy. ESM: asynchronous, static analysis (compile-time), live bindings. ESM tốt hơn vì: (1) **Tree shaking** — bundler loại unused code (CJS không thể). (2) **Static analysis** — detect errors sớm. (3) **Browser-native** support. (4) **Top-level await** (ESM only). Node.js đang migrate sang ESM → đây là future.

**Q: `??` vs `||` — khác gì?**

> A: `||` trả right-hand side khi left-hand side **falsy** (false, 0, "", null, undefined, NaN). `??` trả right-hand side CHỈ khi left-hand side **null hoặc undefined**. `0 || 5 = 5` nhưng `0 ?? 5 = 0`. Dùng `??` khi 0, "", false là giá trị hợp lệ.

**Q: Map vs Object — khi nào dùng Map?**

> A: Dùng Map khi: (1) Key không phải string/symbol (objects, numbers, etc.), (2) Cần biết .size, (3) Frequent add/delete, (4) Cần iteration order. Dùng Object khi: JSON serialization, simple config, prototype methods needed.
