## 1. Interview Pipeline Overview

```
TYPICAL BIG TECH PIPELINE:

  Phone Screen (1h)
    │ Coding + brief behavioral
    ▼
  On-site / Virtual Loop (4-5 rounds × 45-60 min)
    │
    ├── Round 1: Coding (DSA) ────────── 45 min
    │   └── 1-2 problems, optimize brute → optimal
    │
    ├── Round 2: System Design ──────── 45-60 min
    │   └── Design Twitter/Uber/YouTube (see §5.4)
    │
    ├── Round 3: Domain Deep Dive ───── 45-60 min
    │   └── Node.js internals, DB, past projects
    │
    ├── Round 4: Behavioral ──────────── 45 min
    │   └── STAR stories (see §9.1)
    │
    └── Round 5: Bar Raiser (optional)── 45 min
        └── Mixed technical + behavioral + culture
    │
    ▼
  Hiring Committee Decision

  ⚠️ 1 round "No Hire" = likely reject
  → Phải PASS tất cả rounds!
```

## 2. Round 1: Coding (DSA)

```
Framework (45 min):

  1. CLARIFY (3-5 min):
     Input/output? Constraints? Edge cases?
     "Can the array contain negative numbers?"
     "What if the input is empty?"

  2. APPROACH (5 min):
     Nói approach (brute force TRƯỚC)
     "My initial thought is O(n²) with nested loops..."
     "But I think we can optimize with a HashMap to O(n)"
     → Interviewer confirm trước khi code!

  3. CODE (20 min):
     Clean code, meaningful variable names
     Talk while coding: "Here I'm iterating..."
     Methods/functions (not one giant block)

  4. TEST (5-10 min):
     Walk through with example input
     Edge cases: empty, single element, duplicates
     "Let me trace through with input [2, 7, 11, 15], target=9"

  5. OPTIMIZE (5 min):
     Time/Space complexity analysis
     "This is O(n) time, O(n) space"
     Can we optimize space? Trade-offs?
```

```
Common Patterns cần nhớ:

  Two Pointers:     sorted array problems
  Sliding Window:   subarray/substring problems
  HashMap:          frequency, two sum, anagrams
  Stack:            parentheses, monotonic stack
  BFS/DFS:          graph, tree traversal
  Binary Search:    sorted array, search space
  Dynamic Prog:     optimization, counting problems
  Backtracking:     permutations, combinations, subsets
  Heap:             top K, merge K sorted lists
  Union-Find:       connected components, cycle detection

  80% bài interview thuộc 1 trong 10 patterns trên!
```

## 3. Round 2: System Design

```
Framework (xem §5.4 chi tiết):

  1. REQUIREMENTS (5 min):
     Functional: "Users can create short URLs"
     Non-functional: "100K writes/day, 10B reads/day"
     Back-of-envelope: QPS, storage, bandwidth

  2. HIGH-LEVEL (10 min):
     Draw: Client → LB → API → Service → DB/Cache
     Data model: key entities
     API: endpoints

  3. DEEP DIVE (20 min):
     Interviewer chọn component
     Database: schema, indexing, sharding
     Cache: strategy, invalidation
     Scale: bottleneck → solution

  4. WRAP UP (5 min):
     Trade-offs explain
     Monitoring, error handling
     Future improvements

  ⚠️ Senior expectation: DRIVE the conversation!
  → Đừng chờ interviewer hỏi, chủ động nêu trade-offs
  → "I chose X over Y because in this context..."
```

## 4. Round 3: Domain Deep Dive

```
Node.js Senior Questions (phải trả lời SÂUP):

Event Loop:
  "Explain the event loop phases"
  → 6 phases: timers→pending→idle→poll→check→close
  → process.nextTick vs setImmediate vs setTimeout(0)
  → Tại sao blocking main thread = disaster

Streams:
  "How does backpressure work?"
  → Readable highWaterMark, Writable drain event
  → pipe() handles automatically

Performance:
  "How would you debug a memory leak?"
  → heap snapshot, --inspect, clinic.js
  → Common causes: event listeners, closures, global cache

Database:
  "Explain MVCC in PostgreSQL"
  → xmin/xmax, snapshot isolation, VACUUM

Architecture:
  "Walk me through a project you designed"
  → Architecture diagram
  → Tại sao chọn tech stack này
  → Trade-offs và lessons learned
  → What would you do differently?

  ⚠️ "What would you do differently?" = KEY question!
  → Shows growth, self-awareness
  → "Tôi sẽ tách service X sớm hơn vì bottleneck..."
```

## 5. Round 4: Behavioral

```
Xem §9.1 chi tiết.

Quick tips:
  ✅ 10+ stories prepared (STAR format)
  ✅ 2 min per story (not 5 min!)
  ✅ Measurable results: "giảm 90%", "tiết kiệm $8K"
  ✅ "I" not "we": your specific contribution
  ✅ Research company values → map stories
```

## 6. Preparation Timeline

```
12-WEEK PLAN:

Week 1-4: FOUNDATIONS
  ├── DSA: 2-3 problems/day (LeetCode medium)
  ├── Patterns: Two Pointers, Sliding Window, HashMap
  ├── Review: Phase 0-1 (CS fundamentals, JavaScript)
  └── Weekly mock interview (coding)

Week 5-8: DEEP DIVE
  ├── DSA: 1-2 problems/day (medium + hard)
  ├── System Design: 2 designs/week (§5.4)
  ├── Review: Phase 2-5 (Node.js, DB, API, System Design)
  └── Weekly mock interview (system design)

Week 9-10: DOMAIN + BEHAVIORAL
  ├── DSA: maintain 1/day
  ├── Node.js deep dive questions
  ├── Prepare 10+ STAR stories
  ├── Review: Phase 6-8 (DevOps, Security, Testing)
  └── Mock behavioral interview

Week 11-12: POLISH
  ├── Full mock interview (all rounds)
  ├── Review weak areas
  ├── Company-specific prep
  └── Rest before interview!

Daily routine:
  Morning: 1 LeetCode problem (45 min)
  Evening: 1 topic review (30 min)
  Weekend: 1 mock interview + 1 system design
```
