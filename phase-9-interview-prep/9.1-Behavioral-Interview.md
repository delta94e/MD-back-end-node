## 1. STAR Method

```
STAR = framework trả lời behavioral questions

S — Situation:  BỐI CẢNH — project gì, team size, timeline
T — Task:       NHIỆM VỤ — role của bạn, mục tiêu cần đạt
A — Action:     HÀNH ĐỘNG — bạn LÀM GÌ (cụ thể, step by step)
R — Result:     KẾT QUẢ — measurable impact (số liệu cụ thể!)

  ⚠️ Lỗi phổ biến:
  ❌ Nói chung chung: "Team làm tốt"
  ❌ Không có số liệu: "Cải thiện performance"
  ❌ Dùng "chúng tôi" thay vì "tôi"

  ✅ Nói cụ thể: "TÔI đề xuất thêm Redis cache"
  ✅ Có số liệu: "Giảm response time từ 2s xuống 200ms (90%)"
  ✅ Nói về bạn: "Tôi lead initiative, coordinate 3 teams"
```

## 2. 6 Chủ Đề Cần Chuẩn Bị

### Leadership

```
"Tell me about a time you led a project"
"How do you mentor junior developers?"

Ví dụ STAR:
S: E-commerce platform, 500K users, team 5 developers
T: Tech lead cho migration từ monolith sang microservices
A: Tôi thiết kế architecture (bounded contexts), tạo RFC document,
   setup CI/CD pipeline, pair programming với juniors,
   weekly tech talks để upskill team
R: Migration hoàn thành 2 weeks sớm deadline,
   deploy time giảm từ 2h xuống 15 min,
   0 production incidents trong 3 tháng đầu
```

### Conflict Resolution

```
"Tell me about a disagreement with teammate"
"How do you handle technical disagreements?"

Framework:
  1. LISTEN: hiểu perspective của họ (paraphrase lại)
  2. DATA: dùng data/evidence thay vì opinion
  3. PROTOTYPE: nếu không đồng ý → build small POC, let data decide
  4. ESCALATE: nếu cần → involve tech lead (constructively)
  5. COMMIT: một khi decided → fully commit dù không phải ý mình

  ⚠️ NEVER: "Tôi đúng, họ sai"
  ✅ ALWAYS: "Chúng tôi có perspectives khác nhau, tôi đề xuất X vì..."
```

### Failure & Incident Response

```
"Tell me about a time something went wrong"
"Describe a production incident you handled"

Ví dụ STAR:
S: Payment service outage, 3AM, affecting 10K users
T: On-call engineer, cần restore service ASAP
A: 1. Triage: check monitoring (Grafana) → DB connection pool exhausted
   2. Mitigate: increase pool size, restart pods
   3. Fix: identify root cause = N+1 query in new feature
   4. Postmortem: document timeline, root cause, action items
   5. Prevent: add query complexity limits, load testing in CI
R: Service restored in 25 min (SLA: 30 min)
   Postmortem led to 3 process improvements
   No similar incident in 6 months after

⚠️ Key: KHÔNG đổ lỗi! Focus on learning + improvement
```

### Impact

```
"What's your biggest technical achievement?"
"How did you improve system performance?"

Tips: LUÔN có số liệu!
  ✅ "Giảm API latency từ 2s → 200ms bằng caching + query optimization"
  ✅ "Giảm AWS bill 40% ($8K/month) bằng right-sizing instances"
  ✅ "Tăng deployment frequency từ 1/week lên 5/day với CI/CD automation"
  ✅ "Code review turnaround từ 2 days → 4 hours"
```

### Communication

```
"How do you explain technical concepts to non-technical stakeholders?"

Framework:
  1. Analogy: database index = mục lục sách
  2. Impact first: "điều này giảm down time 99%"
  3. Visual: diagram, không phải code
  4. Ask: "Bạn muốn tôi đào sâu phần nào?"
  5. Written: RFC, design doc (async communication)
```

### Growth Mindset

```
"How do you stay current with technology?"
"Tell me about learning a new technology quickly"

Answer framework:
  1. Specific example: "Tôi learn Kubernetes cho project X"
  2. Method: "documentation → tutorial → POC → production"
  3. Share: "Sau đó tôi present cho team trong tech talk"
  4. Outcome: "Deploy time giảm, team productivity tăng"
```

## 3. Story Bank Template

```
Chuẩn bị ít nhất 10 stories, mỗi story cover 2-3 themes:

| # | Story (1 line) | Themes | Impact |
|---|----------------|--------|--------|
| 1 | Microservice migration | Leadership, Technical | Deploy 2h→15min |
| 2 | Production outage 3AM | Failure, Ownership | Restored 25min |
| 3 | Disagree on tech stack | Conflict, Data-driven | POC proved approach |
| 4 | Mentor junior dev | Mentorship, Growth | Junior promoted 6mo |
| 5 | Reduce AWS cost | Impact, Initiative | Save $8K/month |
| 6 | ... | ... | ... |

→ 1 story có thể dùng cho NHIỀU questions!
→ Practice nói thành tiếng (2 min per story, không quá 3 min)
```

## 5. Red Flags & Green Flags

```
❌ RED FLAGS (interviewer sẽ reject):
  - Blame others: "teammate code kém nên bug"
  - No metrics: "cải thiện performance" (bao nhiêu?)
  - Vague: "team làm tốt" (BẠN làm gì?)
  - Negative: "manager không support"
  - No learning: failure story nhưng không nói lessons

✅ GREEN FLAGS:
  - Ownership: "TÔI đề xuất, TÔI implement"
  - Metrics: "giảm 90%, tiết kiệm $8K"
  - Learning: "sau incident, tôi thêm process X"
  - Empathy: "tôi hiểu teammate concern vì..."
  - Growth: "tôi chưa biết X, nên tôi learn bằng..."
```
